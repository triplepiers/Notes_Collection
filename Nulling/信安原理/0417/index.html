    Executable Infectors

    • 可执行程序和文本文件都有一个 header，但是 exe 有二进制的程序，读完 header 后会跳到入口处开始执行

    • 病毒会插到原本程序开始的地方（读完 header 开始执行病毒）

    • 杀毒软件基于“病毒库”进行杀毒，逐个扫描可执行文件中是否包含病毒库中的代码来认定当前程序是否存在病毒（可以单独把病毒代码摘下来）

      • 因为是二进制代码，所以是 memcompare（但一般会搞一个 DFA 出来）
      • 前提「非运行时检查」—— 基于 signature 杀毒
        • 我有病毒库
        • 保证病毒插入在文件的 前N个字节（全文扫描代价太大了）

    • 几种插入方式

      • overwriting => 覆写，破坏原本的程序（不能用了）

      • 接在开头

        前两个在开头，容易被检测

      • 拆成两端 beginning & end（加 jump）=> 躲避检测

      • intersperse（分块，开头jump to begin，中间 jump 到下一段）
      • companions（在 exe 中调用病毒）

    Multipartite Viruses

    有两部分： 1. 感染 boot sector infector 2. 感染 exe files

    即便杀死了所有 exe 中的病毒，下一次启动还是会从启动区带入病毒

    TSR Viruses

    驻留内存的病毒（感染操作病毒的核心进程区域） => 在 exe 中找不出来

    Stealth Viruses

    隐身病毒 -> 隐藏自己的感染信息（有“木马”特征）

    • 检查 length -> 返回未感染时的长度
    • open file -> 把自己干掉，关掉的时候再次感染
    • load file 去执行 -> 给出被感染的文件

    杀毒软件应该用一个非常底层的API去检查文件系统

    Encrypted Viruses

    更难根据 signature 检查出来

    除了密钥、解密程序之外的部分加密 我们在感染不同程序时使用不同 key -> 静态情况下每一个都不一样 => 只能查解密段代码

    Virus Emulation

    通过 random 产生 key & 解密算法不断变种

    Polymorphic Viruses 多态

    随机不用一些代码 / 用不同的 algorithm 产生相同的运行结果

    Macro Viruses

    可以感染 exe / data file

    Worm

    • 独立的可执行程序
    • 会疯狂 copy 自己（传播本身就是恶意行为 -> 可能把 BW 耗尽）

    Defenses of Malicious Code

    There is NO algorithm to detect ALL possible malicious code

    虽然没有完美的解决方案，但是 - Signature-based antivirus - Look for known patterns in malicious code - Always battle with attacker - Greate Business Model！ - Virus Analysis - by human export - aotomated by Immune System（每个人上报，统一进行分析） - 防止 Signature Extraction @ VAC - 区分 data & instruction（严格禁止修改自己的code） - 限制对进程对文件的访问 - Watchdog / Sandbox - 禁止 sharing - 探测 altering of file - 探测 action beyond specification 对每一个 seq 计算一个 checksum，加密结果并存储 执行前重新计算 checksum 并比较 -> 不同则不予执行 - 分析 statistical characteristics

    Botnet 僵尸网络

    Network of autonomous programs capable of acting on instructions

    用于进行以下攻击： - DDoS - Spam & Click Fraud - 新 exploits / worm 的跳板

    一般利用漏洞在僵尸机器上执行一个 short program => 随后，这个 program 会下载并安装真正的 bot 软件

    bot 软件将使得防火墙和杀毒软件失效，并让机器加入聊天频道 Master 将在频道中发布指令操纵整个僵尸网络发起攻击

    DoS 基于不对称性

    • ICMP - Smurf
    • Ping Flood
    • SYN Flood
    • UDP Flood

    DDoS 不可防御

    首先有一个僵尸网络 - 不能被软件防御 - 不需要 IP Spoofing => 反正是 bot 发不是本人发
