    0 Preparations in C

    0.1 Pointer

    C 语言的精髓在于指针

    本节使用了经典的 Swap() 函数作为 intro，介绍了参数传值与传址的区别，从而指出了指针的重要性。

    • 事实上，C 语言的 所有函数参数都是 pass by VALUE，传递的指针本身也是一个值拷贝

      => 但我们可以通过这个拷贝提供的地址找到并修改其所指向的原始数据副本（Caller's Active Record / Global Variable）

    • 在调用函数时，C 编译器将每个参数作为 expression 进行计算，并将其压入 Runtime Stack 的 Callee 栈帧中

    下图是在 main() 中调用函数 swap(3,4) 时的 Runtime Stack 状态：

    1. 指针声明，语法为 type* ptrName

      int*  ptr; // 指针类型决定了 C 将如何处理所指向的地址（但所有指针自身长度一致）
void* ptr; // void* 可以指向任何类型

    2. 取值运算与取地址运算

      int i = 100; int* iPtr;
iPtr = &i;   // iPtr = addr(i)             取地址运算符允许我们获取指定对象的内存地址
*Ptr = 200;  // Mem[iPtr] = 200 => i = 200 取值运算允许我们 读取/修改 对应地址中的数据

    3. 使用指针实现参数传址

      还是 Swap()
      void Swap(int* a, int* b) {
    int tmp;
    tmp = *a;
    *a  = *b;
    *b  = tmp;
}

void main() {
    int i = 3, j = 4;
    Swap(&i, &j);      // 调用时需要传址
}

    4. 空指针 NULL，在 C 中被宏替换为 0

    0.3 动态内存分配

    好麻烦啊 😟

    • C 程序中的变量可能被分配在三种内存区间中：

      • Global Data Section
      • Runtime Stack：局部变量（执行时自动分配）
      • Heap：动态分配空间（根据代码分配和销毁）

    • 动态分配内存空间的工作流程大致如下：

      1. 在 执行时，进程向 Memory Allocator 发起了申请（携带了所需 Byte 数量）
      2. Memory Allocator 将 申请大小的连续内存地址 标为 allocated，同时返回指向起始地址的指针

      若没有显式释放空间，该内存空间将被持续占用

      动态分配空间可以 跨函数访问 -> 共用的 Heap 空间

    变长数组实现

    数组长度取决于输入 numAircraft
    int numAircraft; Flight* planes;
scanf("%d", &numAircraft); // 获取输入

// malloc() 的返回类型为 void*，需要进行强制类型转化
// sizeof() 返回了单个 Flight 结构体需要的 Byte 数量
planes = (Flight*)malloc(sizeof(Flight) * numAircraft);

// 若当前 Heap 中的剩余空间不足以满足需求，malloc() 将返回 NULL
if (planes == NULL) {
    printf("Fail to allocate space.\n")
}

// void* realloc(void* ptr, int size) 支持动态调整 ptr 指向的空间大小
// 当 size == 0 时，与 free 等价（返回 NULL）
planes = (Filght*)realloc(planes, 25);

// 不要忘了手动 free
free(planes);

    • 由于 malloc() 分配的空间是连续的，我们可以进行数组形式的下标访问

      planes[i] 可以让我们读取第 i+1 架飞机的详细信息

    1 Stack

    因为 Call-Return Mechanism 是用 Stack 实现的，所以 Patt 这里拿来当 intro 了

    1.1 子程序调用

    • 子程序 Subroutine

      由 库 Library 提供的程序片段被称为 “子程序”（也可以叫 “函数”）

      我们可以通过调用子程序来减少代码段的大量重复

    • 调用/返回机制 Call/Return Mechanism

      • 调用过程

        • Caller 向 Runtime Stack 中压入实参
        • R7 = PC+1 记录返回地址（函数调用的下一条指令）
        • PC = funcAddr PC 指向子程序起始位置

        • 在 LC-3 中对应 JSR(R) (0100) 指令：

          opcode mode offset desc
          0100 1 11 bit offset addr = PC + offset
          0100 0 00 xxx 000000 addr = Reg[base]

      • 返回过程

        PC = R7 PC 指向返回地址

        在 LC-3 中就是一条 JMP R7 的无条件跳转指令（相当于 return）

    • 保存/恢复 寄存器 Save/Restore

      当后续指令会覆盖寄存器中的值，但还需要使用该值时（一些 out-active），我们需要保存寄存器中的数据至 Memory

      Callee-Saved Registers

      这一情形在子程序调用中尤为常见：

      • 我们会在子函数开始前将这些值保存到 Mem
      • 在返回前将其从 Mem 恢复至原来的 Reg 中（在 Caller 看来好像没有变化）

      此外，寄存器 R7 是 Caller-Saved 的（因为调用时会被返回地址覆盖）

    Runtime Stack

    在 LC-3 中，我们使用 R6(sp) 指向栈顶，R5(fp) 指向当前活跃函数的栈帧

    • @ Assembly-Level，函数只是：

      • 一连串可以通过 JSR 调用执行的指令序列
      • 以 RET 收尾，使得控制流返回 Caller

    • Stack Frame / Activation Record 栈帧 / 活动记录

      每当我们调用函数，就在 Runtime Stack 中为该函数实例创建一个活动记录，用于储存其参数、局部变量、Bookkeeping Info（Caller's fp, Return Value & Return Address）

      当函数调用结束时，我们就将该活动记录从 Runtime Stack 中弹出

    编译过程

    对于 C 指令 w = Volt(w, 10) LC-3 编译器将执行以下动作：

    1. 将两个实参 10 & w 压入 R6 指向的 Runtime Stack 栈顶

      Stack Protocol 要求形参 自右向左 入栈

      假设 w 存储在 R5 指向的位置
      # Push(10)
AND R0, R0, #0;
ADD R0, R0, #10;
ADD R6, R6, #-1;  sp--
STR R0, R6, #0;   *sp = 10

# Push(w)
LDR R0, R5, #0;   load(w)
ADD R6, R6, #-1;
STR R0, R6, #0;

    2. 通过 JSR 指令使得 Control Flow 跳转至 Volt() 起始位置

      JSR Volt

    3. 执行 Callee Function

      R5(fp) 指向首个 Local Variable

      1. Set the stage

        • 在栈顶为 返回值 预留空间
        • 将 R7 中存储的 返回地址 入栈（防止嵌套调用导致的覆盖）
        • 将 R5 中存储的 Caller's fp 入栈
        • 修改 R6(sp) 为 Callee 中的局部变量分配空间，修改 R5(fp) 使其指向 Calllee 的栈帧

        Volt:   # Set the stage
        ADD R6, R6, #-1;  为 return Value 预留的空间
        ; Push(return address)
        ADD R6, R6, #-1;
        STR R7, R6, #0;
        ; Push(caller's fp)
        ADD R6, R6, #-1;
        STR R5, R6, #0;
        ; 让 fp 指向 Volt 的栈帧
        ADD R5, R6, #-1;
        ; 为 Volt 的局部变量预留空间
        ADD R6, R6, #-2;

      2. End the Callee Function

        • （如果存在）将返回值写入栈帧的预留位置
        • 将所有局部变量弹出（我们可以逐个弹出，但直接令 sp = fp+1 更加高效）
        • 从 Runtime Stack中恢复 Caller‘s fp & Return Address
        • 使用 RET 指令返回 Caller
        # 此处我们认为返回值为局部变量 k -> 恰好是 R5 指向的位置
LDR R0, R5, #0; 
STR R0, R5, #3;  将返回值 k 写到预留 slot（这个位置固定为 fp+3）

ADD R6, R5, #1;  Pop(local variables)

LDR R5, R6, #0;  Restore(caller's fp)
ADD R6, R6, #1;  Pop

LDR R7, R6, #0;  Restore(return address)
ADD R6, R6, #1;  Pop

RET;
# 此时 return Value & params 仍在栈中

    4. 返回 Caller Function

      在真正返回执行 Caller Function 前，我们仍有一些工作需要完成：

      • （如果存在）从 Runtime Stack 中弹出 Return Value
      • 从 Rumtime Stack 中弹出所有实参
      针对 w = Volt(w, 10) 这一实例
      JSR Volt;

# this is where we begin
LDR R0, R6, #0;   R0 = *sp = return Val
STR R0, R5, #0;   w = return Val
ADD R6, R6, #1;   Pop(return Val)

ADD R6, R6, #2;   Pop(args) -> 这部分的数字取决于实参列表的具体长度

    1.3 栈（本体）

    Stack 是一种抽象数据类型（只要实现了 FILO 就行），它描述了存储机制而非具体的实现方法

    • Stack 有两个必要的方法：Push（向栈顶压入元素），Pop（从栈顶弹出元素）
    • 我们总是访问栈顶的元素，该元素一般被标识为 Top

    内存中的栈实现

    内存中的栈通常由一段 连续 的内存空间 + 栈顶指针 sp 构成：

    由 Stack 占用的内存空间不允许通过其他机制访问（即便该位置暂时空闲）

    • sp 是一个寄存器，始终指向栈顶元素 且 在 Push / Pop 时均被更新

      在 LC-3 中，通常使用 R6 作为 sp 寄存器，起始值 == -EMPTY

    • 栈中的剩余数据在 Push / Pop 时不需要移动（只要修改 sp ）

    • 两种溢出

      • 下溢 Underflow：对空栈执行 Pop
      • 上溢 Overflow：对满栈执行 Push

    • 跳转至单独的溢出处理函数 UNDERFLOW / OVERFLOW 并不是一个理想的操作，不如记录执行 成功/失败 后直接返回 Caller：

      我们将执行信息存储到 R5 中（ 0 = success; 1 - fail），通过读取 R5 即可知道 Pop 是否执行成功

    栈操作的 LC-3 实现
    # Stack 由 高地址 -> 低地址 增长

# 入栈
PUSH:      ;                  Callee-Save
           ST R2, Save2;      M[Save2] = R2
           ST R1, Save1;      M[Save1] = R1
           ;                  上溢检查
           LD R1, MAX;        R1 = MAX = -addr(stack_full)
           ADD R2, R6, R1;    R2 = sp + MAX, == 0 时为满栈
           BRz Failure;       R2 == 0，产生 overflow
           ;                  栈未满，压入程序
           ADD R6, R6, #-1;   sp（R6） 向低地址增长
           STR R0, R6, #0;    R0 入栈（sp 指向的位置）
           BRnzp Success;     成功返回片段
MAX:       .FILL xC005;       MAX = -addr(stack_full) = -x3FFB = xC005

# 出栈
POP:       ;                  Callee-Save
           ST R2, Save2;      M[Save2] = R2
           ST R1, Save1;      M[Save1] = R1
           ;                  下溢检查
           LD R1, EMPTY;      R1 = EMPTY = -addr(stack_bottom)
           ADD R2, R6, R1;    R2 = sp + EMPTY，== 0 时为空栈
           BRz Failure;       R2 == 0，产生 underflow
           ;                  栈非空，弹出程序
           LDR R0, R6, #0;    将栈顶元素弹出至 R0
           ADD R6, R6, #1;    sp（R6） 向高地址收缩 
           BRnzp Success;     成功返回片段
EMPTY:     .FILL xC00;        EMPTY = -addr(stack_bottom) = -x4000 = xC000

# 统一的失败返回片段
Failure:   AND R5, R5, #0;    初始化，令 R5 = 0
           ADD R5, R5, #1;    R5 = 1，表示执行失败
           RET;               直接返回 Caller

# 统一的成功返回片段
Success:   ;                  Callee-Restore
           LD R1, Save1;
           LD R2, Save2;
           AND R5, R5, #0;    R5 = 0，表示执行成功
           RET;               返回 Caller

# 初始化内存空间
Save1:     .FILL x0000;
Save2:     .FILL x0000;

    上述片段使用了 R1，R2，R5：

    • Callee-Saved：R1，R2 在子程序结束后不再使用 —— 由子程序恢复
    • Caller-Saved：R5 在结束后先用于判断是否成功，再恢复原值 —— 由调用者自行恢复

    2 Array

    数组是一串 连续的内存空间，支持通过 下标访问 特定位置的元素

    2.1 数组声明

    在 C 语言中，我们按照如下语法声明一个数组：

    Array 由 低地址 -> 高地址 分布
    int arr1[10];  // type arrName[Size]

// 数组长度不能是变量（以下操作非法）
int len;
int arr2[len]; // 编译时无法确定分配多大的空间

// 但可以是一个宏（以下操作合法）
#define LEN 10
int arr3[LEN]; // 本质是 CONST

    当 arr[] 作为 局部变量 时，该空间会从 Runtime stack 中分配

    2.2 下标访问

    因为数组空间是连续的，因此可以快速通过下标计算指定元素的地址：

    \[ \&arr[index] = base + index * width \]

    那么对于 C 语言片段：

    arr[x+1] = arr[x] + 2;

    我们可以通过如下的 LC-3 汇编实现：

    # size(arr) = 10，且 R5 = &a[9]     -> 最高地址
# 假设 int x 被存储在 a[0] 的前一个位置 -> &x = &a[9] - 10 = R5 - 10

LDR R0, R5, #-10; R0 = x
ADD R1, R5, #-9;  R1 = &a[0]
ADD R1, R0, R1;   R1 = &a[x]
LDR R2, R1, #0;   R2 = a[x]
ADD R2, R2, #2;   R2 = a[x] + 2
LDR R0, R5, #-10; R0 = x（感觉有点重复？）
ADD R0, R0, #1;   R0 = x+1
ADD R1, R5, #-9;  R1 = &a[0]
ADD R1, R0, R1;   R1 = &a[x+1]
STR R2, R1, #0;   a[x+1] = a[x] + 2

    2.3 参数传递

    C 语言中的数组参数 默认传址

    // 参数声明
int Average(int a[]);

// 参数传递
int arr[10];
Average(arr);   // 传数组名就可以了

// 值得注意的是，我们可以认为 arr 是 int* 类型
int* ptr = arr; // 合法

// 但 arr 作为标识符不可被更改
int arr1[10];
arr = arr1;    // 非法

    2.4 String 的数组实现

    鉴于 C 本身并不支持 string 类型，我们可以用 char[] 进行模拟：

    本质是存储了 零扩展 ASCII 的 int[]

    • 实际定义的数组长度应为 len(str) + 1 ，并手动令 ch[size] = '\0'

    • 数组名本身指向了字符串的起始位置，我们可以用 *(str + i) 访问 str[i]

    2.4.2 LC-3 判断字符串相等

    Success: R5 = 0
    STRCMP:    ;                   Callee-Save
           ST R0, SaveR0;      R0 = &str1[0]
           ST R1, SaveR1;      R1 = &str2[0]
           ST R2, SaveR2;
           ST R3, SaveR3;
           ;
           AND R5, R5, #0;     R5 = 0，默认 Equal
NEXTCHAR:  LDR R2, R0, #0;     R2 = *R0 = str1[idx]
           LDR R3, R1, #0;     R3 = *R1 = str2[idx]
           BRnp COMPARE;       str[2] 还没结束（不是'\0'），继续比较
           ADD R2, R2, #0;     R2 = R2 + 0
           BRz RESTORE;        R2 = 0 & R3 = 0，两个字符串均结束
COMPARE:   NOT R2, R2;
           ADD R2, R2, #1;     R2 = !R2 + 1 = -str1[idx]
           ADD R2, R2, R3;     R2 = -str1[idx] + str2[idx]
           BRnp NEQ;           R2 != 0，当前字符不等
           ADD R0, R0, #1;     R0 = &str1[idx+1]
           ADD R1, R1, #1;     R1 = &str2[idx+1]
           BRnzp NEXTCHAR;     检查下一个 char
NEQ:       ADD R5, R5, #1;     R5 = 1，Not Equal
RESTORE:   LD R0, SaveR0;      Callee-Restore
           LD R1, SaveR1;
           LD R2, SaveR2;
           LD R3, SaveR3;
           RET;

SaveR0:    .BLKW 1
SaveR1:    .BLKW 1
SaveR2:    .BLKW 1
SaveR3:    .BLKW 1

    2.4.3 使用 char 数组表示整数

    一般用于大数运算

    依次检查 x30 <= ch[idx] <= x39

    Success: R5 = 0
    # Input: R0 = &ch[0] 初始值为 ch[] 的起始位置

TEST_INT:   ;                Callee-Store
            ST R1, SaveR1
            ST R2, SaveR2
            ST R3, SaveR3
            ST R4, SaveR4
            ;
            AND R5, R5, #0;  R5 = 0，默认符合
            LD R2, ASCII_0;  R2 = -asc(0)
            LD R3, ASCII_9;  R3 = -asc(9)
NEXT_CHAR:  LDR R1, R0, #0;  R1 = ch[idx]
            BRz SUCESS;      完成比较（检查到 `\0`）
            ADD R4, R1, R2;  R4 = asc(ch[i]) - asc(0)
            BRn BAD;         asc(ch[i]) < asc(0)
            ADD R4, R1, R3;  R4 = asc(ch[i]) - asc(9)
            BRp BAD;         asc(ch[i]) > asc(9)
            ADD R0, R0, #1;  R0 = &ch[idx++]
            BR  NEXT_CHAR;
FAIL:       ADD R5, R5, #1;  R5 = 1, 不符合
SUCESS:     ;                Callee-Restore
            LD R4.SaveR4;
            LD R3.SaveR3;
            LD R2.SaveR2;
            LD R1.SaveR1;
            RET;

    2.6 C 语言数组的不足

    1. 未提供下标越界保护

      当使用 arr[idx] 访问数组元素时，C 并不会检查 idx 是否超出上下界

      => 不报错，但是会取到一些怪玩意儿

      => 因为不检查，C 也拥有了 更快的执行速度

    2. 固定的数组长度

      因为 C 要求在编译时确定数组大小，所以不支持用变量声明数组长度

      常见的解决方法有：估计并申请足够大的空间 / 动态内存分配

    2.7 应用

    2.7.1 数组反转

    原地反转
    int lenStr(char str[]) {
    int len = 0;
    while(str[len] != '\0') len++;
    return len;
}
void swapChar(char* ch1, char* ch2) {
    char tmp;
    tmp = *ch1; *ch1 = *ch2; *ch2 = tmp;
}
void Reverse(char str[]) {
    int len = lenStr(str);
    for (int i = 0 ; i < len/2 ; i++) {
        swapChar(&str[i], &str[len-(i+1)]);
    }
}

    2.7.2 插入排序

    #define MAX_LEN 10

// 升序排列
void insertionSort(int list[]) {
    // 默认 len(1) 数组有序，所以从第二个元素开始插入
    for (int i = 1 ; i < MAX_LEN ; i++) {
        int cur = list[i];
        // 自右向左遍历 -> 依次后移
        int j;
        for(j = i-1 ; j>=0 && list[j]>cur ; j--) {
            list[j+1] = list[j];
        }
        // 在当前位置插入新元素
        list[j+1] = cur;
    }
}

void main() {
    int nums[MAX_LEN];
    for(int i = 0 ; i < MAX_LEN ; i++) scanf("%d", &nums[i]);
    insertionSort(nums);
}

    3 Linked List

    不同于数组，链表的每个元素在内存空间中 可能是不连续的，因此 不允许随机访问：

    • 我们称链表中的每一个元素为 Node，存储了一个 Unit 的数据（一般是由 struct 定义的结构体）

    • 相邻节点通过 指针 node* next 相连，只能 顺序访问 -> 查找需要 O(N)

      好处是 INSERT / DELETE 只需要 O(1)

    • 我们用 head pointer 指向链表的 首个节点，并且令 tail 节点的 next = NULL

    3.2 Insert

    // 保证按照 ID 升序排列
int insert(node* neo, node** list) {
    node* pre = NULL;
    node* cur = *list;           // 指向首节点
    int cmp;

    while(cur != NULL) {         // 还未到达末尾
        cmp = strcmp(neo->id, cur->id);
        if (cmp == 0) {
            return -1;           // 已存在
        } else if (cmp < 0) {     // 插入当前位置
            neo->next = cur;
            if (pre == NULL) {
                *list = neo;     // Add at head
            } else {
                pre->next = cur; // Add at middle
            }
            return 0;            // 成功插入
        } else {
            pre = cur;
            cur = cur->next;
        }
    }
    neo->next = NULL;
    if (pre->next == NULL) {
        *list = neo;             // empty list
    } else {
        pre->next = neo;         // Add at tail
    }
    return 0;
}

    3.3 Delete

    int Delete(char* id, node** list) {
    node* pre = NULL;
    node* cur = *list;
    int cmp;

    while(cur != NULL) {
        cmp = strcmp(id, cur->id);
        if (cmp == 0) {
            if (pre == NULL) {
                *list = cur->next;      // Drop @ head
            } else {
                pre->next = cur->next;  // Drop @ middle/tail
            }
            free(cur);
            return 0;
        } else if (cmp < 0) {
            return -1;                  // Don't Exist
        } else {
            pre = cur;
            cur = cur->next;
        }
    }

    return -1;                          // Don't Exist
}
