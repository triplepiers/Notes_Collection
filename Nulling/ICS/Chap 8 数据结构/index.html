    Chap 8 数据结构

    1 Stack

    因为 Call-Return Mechanism 是用 Stack 实现的，所以 Patt 这里拿来当 intro 了

    1.1 子程序调用

    • 子程序 Subroutine

      由 库 Library 提供的程序片段（函数）被称为 “子程序”

      我们可以通过调用子程序来减少代码段的大量重复

    • 调用/返回机制 Call/Return Mechanism

      • 调用过程

        • reg[7] = PC+1 记录返回地址（从函数调用的下一条指令开始执行）

        • PC = addr(funcName) 下一个周期开始执行调用的函数

        • 在 LC-3 中对应 JSR(R) (0100) 指令：

          opcode mode offset desc
          0100 1 11 bit offset addr = PC + offset
          0100 0 00 xxx 000000 addr = Reg[base]

      • 返回过程

        PC = reg[7] 从特定寄存器中获取之前保存的返回地址

        在 LC-3 中就是一条 JMP R7 的无条件跳转指令（相当于 return）

    • 保存/恢复 寄存器 Save/Restore

      当后续指令会覆盖寄存器中的值，但还需要使用该值时（一些 out-active），我们需要保存寄存器中的数据至 Memory

      Callee-Saved Registers

      这一情形在子程序调用中尤为常见：

      • 我们会在子函数开始前将这些值保存到 Mem
      • 在返回前将其从 Mem 恢复至原来的 Reg 中（在 Caller 看来好像没有变化）

      此外，寄存器 reg[7] 是 Caller-Saved 的（因为调用时会被返回地址覆盖）

    1.3 栈（本体）

    Stack 是一种抽象数据类型（只要实现了 FILO 就行），它描述了存储机制而非具体的实现方法

    • Stack 有两个必要的方法：Push（向栈顶压入元素），Pop（从栈顶弹出元素）
    • 我们总是访问栈顶的元素，该元素一般被标识为 Top

    1.3.1 内存中的栈实现

    内存中的栈通常由一段 连续 的内存空间 + 栈顶指针 sp 构成：

    由 Stack 占用的内存空间不能通过其他机制访问（即便该位置暂时空闲）

    • sp 是一个寄存器，始终指向栈顶元素（在 Push / Pop 时均被更新）

      在 LC-3 中，通常使用 r[6] 作为 sp 寄存器，起始值 == -EMPTY

    • 栈中的剩余数据在 Push / Pop 时不需要移动（只要修改 sp ）

    • 下溢出 Underflow：对空栈执行 Pop

    • 上溢出 Overflow：对满栈执行 Push

    • 跳转至单独的溢出处理函数 UNDERFLOW / OVERFLOW 并不是一个理想的操作，不如记录执行 成功/失败 后直接返回 Caller：

      我们将执行信息存储到 R5 中（ 0 = success; 1 - fail），通过读取 R5 即可知道 Pop 是否执行成功

    栈操作的 LC-3 实现
    # Stack 由 高地址 -> 低地址 增长

# 入栈
PUSH:      ;                  Callee-Save
           ST R2, Save2;      M[Save2] = R2
           ST R1, Save1;      M[Save1] = R1
           ;                  上溢检查
           LD R1, MAX;        R1 = MAX = -addr(stack_full)
           ADD R2, R6, R1;    R2 = sp + MAX, == 0 时为满栈
           BRz Failure;       R2 == 0，产生 overflow
           ;                  栈未满，压入程序
           ADD R6, R6, #-1;   sp（R6） 向低地址增长
           STR R0, R6, #0;    R0 入栈（sp 指向的位置）
           BRnzp Success;     成功返回片段
MAX:       .FILL xC005;       MAX = -addr(stack_full) = -x3FFB = xC005

# 出栈
POP:       ;                  Callee-Save
           ST R2, Save2;      M[Save2] = R2
           ST R1, Save1;      M[Save1] = R1
           ;                  下溢检查
           LD R1, EMPTY;      R1 = EMPTY = -addr(stack_bottom)
           ADD R2, R6, R1;    R2 = sp + EMPTY，== 0 时为空栈
           BRz Failure;       R2 == 0，产生 underflow
           ;                  栈非空，弹出程序
           LDR R0, R6, #0;    将栈顶元素弹出至 R0
           ADD R6, R6, #1;    sp（R6） 向高地址收缩 
           BRnzp Success;     成功返回片段
EMPTY:     .FILL xC00;        EMPTY = -addr(stack_bottom) = -x4000 = xC000

# 统一的失败返回片段
Failure:   AND R5, R5, #0;    初始化，令 R5 = 0
           ADD R5, R5, #1;    R5 = 1，表示执行失败
           RET;               直接返回 Caller

# 统一的成功返回片段
Success:   ;                  Callee-Restore
           LD R1, Save1;
           LD R2, Save2;
           AND R5, R5, #0;    R5 = 0，表示执行成功
           RET;               返回 Caller

# 初始化内存空间
Save1:     .FILL x0000;
Save2:     .FILL x0000;

    上述片段使用了 R1，R2，R5：

    • Callee-Saved：R1，R2 在子程序结束后不再使用 —— 由子程序恢复
    • Caller-Saved：R5 在结束后先用于判断是否成功，再恢复原值 —— 由调用者自行恢复

    2 Array

    这一节突发引入了 C 语言？

    数组是一串 连续的内存空间，支持通过 下标访问 特定位置的元素

    2.1 数组声明

    在 C 语言中，我们按照如下语法声明一个数组：

    Array 由 低地址 -> 高地址 分布
    int arr1[10];  // type arrName[Size]

// 数组长度不能是变量（以下操作非法）
int len;
int arr2[len]; // 编译时无法确定分配多大的空间

// 但可以是一个宏（以下操作合法）
#define LEN 10
int arr3[LEN]; // 本质是 CONST

    当 arr[] 作为 局部变量 时，该空间会从 run-time stack 中分配

    2.2 下标访问

    因为数组空间是连续的，因此可以快速通过下标计算指定元素的地址：

    \[ \&arr[index] = base + index * width \]

    那么对于 C 语言片段：

    arr[x+1] = arr[x] + 2;

    我们可以通过如下的 LC-3 汇编实现：

    # size(arr) = 10，且 R5 = &a[9]     -> 最高地址
# 假设 int x 被存储在 a[0] 的前一个位置 -> &x = &a[9] - 10 = R5 - 10

LDR R0, R5, #-10; R0 = x
ADD R1, R5, #-9;  R1 = &a[0]
ADD R1, R0, R1;   R1 = &a[x]
LDR R2, R1, #0;   R2 = a[x]
ADD R2, R2, #2;   R2 = a[x] + 2
LDR R0, R5, #-10; R0 = x（感觉有点重复？）
ADD R0, R0, #1;   R0 = x+1
ADD R1, R5, #-9;  R1 = &a[0]
ADD R1, R0, R1;   R1 = &a[x+1]
STR R2, R1, #0;   a[x+1] = a[x] + 2

    2.3 参数传递

    C 语言中的数组参数 默认传址

    // 参数声明
int Average(int a[]);

// 参数传递
int arr[10];
Average(arr);   // 传数组名就可以了

// 值得注意的是，我们可以认为 arr 是 int* 类型
int* ptr = arr; // 合法

// 但 arr 作为标识符不可被更改
int arr1[10];
arr = arr1;    // 非法

    2.4 String 的数组实现

    鉴于 C 本身并不支持 string 类型，我们可以用 char[] 进行模拟：

    本质是存储了 零扩展 ASCII 的 int[]

    • 实际定义的数组长度应为 len(str) + 1 ，并手动令 ch[size] = '\0'

      否则在 printf("%s", ch) 时可能因为无法检测到结束符而输出错误结果

    2.4.1 降低 Struct 的大小

    同时也支持不定长度的 string 字段存储

    with Array
    typedef struct person {
    char firstName[10];
    char lastName[10];
    int salary;
}Person;

// 在上述情况下，每个人的名字不超过 9 个字符（显然不太妙）
    with Pointer
    typedef struct person {
    char* firstName;
    char* lastName;
    int salary;
}Person;

// 支持任意长度的姓名

    2.4.2 LC-3 判断字符串相等

    Success: R5 = 0
    STRCMP:    ;                   Callee-Save
           ST R0, SaveR0;      R0 = &str1[0]
           ST R1, SaveR1;      R1 = &str2[0]
           ST R2, SaveR2;
           ST R3, SaveR3;
           ;
           AND R5, R5, #0;     R5 = 0，默认 Equal
NEXTCHAR:  LDR R2, R0, #0;     R2 = *R0 = str1[idx]
           LDR R3, R1, #0;     R3 = *R1 = str2[idx]
           BRnp COMPARE;       str[2] 还没结束（不是'\0'），继续比较
           ADD R2, R2, #0;     R2 = R2 + 0
           BRz RESTORE;        R2 = 0 & R3 = 0，两个字符串均结束
COMPARE:   NOT R2, R2;
           ADD R2, R2, #1;     R2 = !R2 + 1 = -str1[idx]
           ADD R2, R2, R3;     R2 = -str1[idx] + str2[idx]
           BRnp NEQ;           R2 != 0，当前字符不等
           ADD R0, R0, #1;     R0 = &str1[idx+1]
           ADD R1, R1, #1;     R1 = &str2[idx+1]
           BRnzp NEXTCHAR;     检查下一个 char
NEQ:       ADD R5, R5, #1;     R5 = 1，Not Equal
RESTORE:   LD R0, SaveR0;      Callee-Restore
           LD R1, SaveR1;
           LD R2, SaveR2;
           LD R3, SaveR3;
           RET;

SaveR0:    .BLKW 1
SaveR1:    .BLKW 1
SaveR2:    .BLKW 1
SaveR3:    .BLKW 1

    2.4.3 使用 char 数组表示整数

    一般用于大数运算

    下面是一个用于检测 char[] 内容是否为 int 的函数：

    依次检查 x30 <= ch[idx] <= x39

    Success: R5 = 0
    # Input: R0 = &ch[0] 初始值为 ch[] 的起始位置

TEST_INT:   ;                Callee-Store
            ST R1, SaveR1
            ST R2, SaveR2
            ST R3, SaveR3
            ST R4, SaveR4
            ;
            AND R5, R5, #0;  R5 = 0，默认符合
            LD R2, ASCII_0;  R2 = -asc(0)
            LD R3, ASCII_9;  R3 = -asc(9)
NEXT_CHAR:  LDR R1, R0, #0;  R1 = ch[idx]
            BRz SUCESS;      完成比较（检查到 `\0`）
            ADD R4, R1, R2;  R4 = asc(ch[i]) - asc(0)
            BRn BAD;         asc(ch[i]) < asc(0)
            ADD R4, R1, R3;  R4 = asc(ch[i]) - asc(9)
            BRp BAD;         asc(ch[i]) > asc(9)
            ADD R0, R0, #1;  R0 = &ch[idx++]
            BR  NEXT_CHAR;
FAIL:       ADD R5, R5, #1;  R5 = 1, 不符合
SUCESS:     ;                Callee-Restore
            LD R4.SaveR4;
            LD R3.SaveR3;
            LD R2.SaveR2;
            LD R1.SaveR1;
            RET;

ASCII_0:    .FILL xFFD0;    -asc(0)
ASCII_9:    .FILL xFFC7;    -asc(9)
SaveR1:     .BLKW 1
SaveR2:     .BLKW 1
SaveR3:     .BLKW 1
SaveR4:     .BLKW 1

    2.5 高维数组

    以二维数组为例，我们可以很简单的声明：

    int arr2d[1080][1902];  // 总大小为 1080*1902

    在内存中的分布如下：

    三维数组
    int data[40][50][60]; // data[0][0][59] 的相邻元素是 data[0][1][0]

    2.6 C 语言数组的不足

    1. 未提供下标越界保护

      当使用 arr[idx] 访问数组元素时，C 并不会检查 idx 是否超出上下界

      => 不报错，但是会取到一些怪玩意儿

      => 因为不检查，C 也拥有了 更快的执行速度

    2. 固定的数组长度

      因为 C 要求在编译时确定数组大小，所以不支持用变量声明数组长度

      常见的解决方法有：估计并申请足够大的空间 / 动态内存分配

    3 Linked List

    不同于数组，链表的每个元素在内存空间中 可能是不连续的，因此 不允许随机访问：

    • 我们称链表中的每一个元素为 Node，存储了一个 Unit 的数据（一般是由 struct 定义的结构体）

    • 相邻节点通过 指针 node* next 相连，只能 顺序访问 -> 查找需要 O(N)

      好处是 INSERT / DELETE 只需要 O(1)

    • 我们用 head pointer 指向链表的 首个节点，并且令 tail 节点的 next = NULL

    3.1 Insert

    // 保证按照 ID 升序排列
int insert(node* neo, node** list) {
    node* pre = NULL;
    node* cur = *list;           // 指向首节点
    int cmp;

    while(cur != NULL) {         // 还未到达末尾
        cmp = strcmp(neo->id, cur->id);
        if (cmp == 0) {
            return -1;           // 已存在
        } else if (cmp < 0) {     // 插入当前位置
            neo->next = cur;
            if (pre == NULL) {
                *list = neo;     // Add at head
            } else {
                pre->next = cur; // Add at middle
            }
            return 0;            // 成功插入
        } else {
            pre = cur;
            cur = cur->next;
        }
    }
    neo->next = NULL;
    if (pre->next == NULL) {
        *list = neo;             // empty list
    } else {
        pre->next = neo;         // Add at tail
    }
    return 0;
}

    3.2 Delete

    int Delete(char* id, node** list) {
    node* pre = NULL;
    node* cur = *list;
    int cmp;

    while(cur != NULL) {
        cmp = strcmp(id, cur->id);
        if (cmp == 0) {
            if (pre == NULL) {
                *list = cur->next;      // Drop @ head
            } else {
                pre->next = cur->next;  // Drop @ middle/tail
            }
            free(cur);
            return 0;
        } else if (cmp < 0) {
            return -1;                  // Don't Exist
        } else {
            pre = cur;
            cur = cur->next;
        }
    }

    return -1;                          // Don't Exist
}
