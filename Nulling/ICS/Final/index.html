19 Fall

  • How many of the 15 LC-3 instructions assert the LD.MDR control signal during its instruction cycle?

    15, because LD.MDR is asserted during FETCH

  • Explain assemble-time error LD, R0, A. error: cannot encode as 9-bit 2's complement intager.

    Label A is too far away from the instruction.

  • Why does MAR contains the value of PC @ the start of the clock cycle, but NOT PC+1 (since 1 is added to the PC during the clock cycle)?

    Flipflops are edge triggered, so PC & MDR change simultaneously.

  • RLC - 1 instructions will be:

    • 8-bit wide
    • use 3-bit for opcode

    • have 4 general purpose registers

    • How many instructions can be in the RLC-1 RSA ?

      \[ 2^3 = 8 \]

    • Can RLC-1 ISA have instruction DR <-- SR1 + SR2 ?

      NO! Each register would take log4 = 2-bit to encode, 3-bit opcode + 3 * 2 bit register takes 9-bit in total ( > 8-bit ).

  • 在以下程序段中，M[x4000] 处存储数据的类型是 Instruction

       .ORIG x3000
    LD R0, E
    JSRR R0 => 调用从 x4000 处开始的函数
    HALT
E   .FILL x4000
   .END

  • 在以下程序段中，main() 开始于 x3000 处，键盘中断处理程序开始于 x1000 处。请问按下按键后，main() 函数会 HALT 吗？（请说明原因）

        # main()
     .ORIG x3000
LOOP  BR LOOP # 这是一个死循环
      HALT
     .END

    # 键盘中断处理程序
     .ORIG x1000
     ST R0, SaveR0
     LDR R0, R6, #0
     ADD R0, R0, #1
     STR R0, R6, #0
     LD R0, SaveR0
     RTI
SaveR0 .BLKW #1
       .END

    YES, because the service routine increments the return address on the stack & popped during RTI

    事实上主程序一直在 LOOP 那一行死循环。按下键盘后，PC+1 将作为返回地址压入栈中 HALT，随后在执行 RTI 时恢复使程序停止。

  • 某个 recursive program 为什么 NOT work？

    There is no BASE CASE. -> 缺少递归终点

  • When does X = 1 ?

    When you are attempting to access privileged memory or a device register from user mode.

    • PSR[15] 用于判断当前进程是否处于 Privileged Mode
    • MAR 用于判断该地址是否处于 系统空间 / device register

21 Fall

  • 一位同学尝试使用以下程序段替代所有的 LD R0, A 语句：

    AND R0, R0, #0
LD R0, A

    How likely will this fix the problem?

    => NOT likely, since AND R0, R0, #0 does NOT change the behavior of the program

  • 下列程序段是否报错 & 哪一行报 Assemble / Runtime 错误？

    1. Assemble Time Error

      AND R0, R0, #1
LEA R1, #0      // R1 = (PC+1) + #0
LDR R2, R0, R1  // 错误，LDR 的第三个参数应该是 imm
HALT

    2. Runtime Error

      AND R0, R0, #0
.FILL xDDDD        // 这一行会被当成指令 -> 非法 opcode
HALT
