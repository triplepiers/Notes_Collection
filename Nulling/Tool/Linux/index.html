    1 包管理系统 apt

    apt 的全称是 Advance Package Tool，是一个用于在 Debian、Ubuntu 或者其他 Linux 发行版上安装和移除软件的自由软件。

    软件源

    apt 软件源列表位于 /etc/apt/sources.list 下，您可以通过以下命令将其列出：

    cat /etc/apt/sources.list | grep -v "#"

    每一条返回记录将遵循以下格式：

    [Archive Type] [Repository URL] [Distribution] [Components]

    您也可以通过以下命令将官方源 http://archive.ubuntu.com/ 替换为国内镜像以提高速度：

    sudo sed -i 's|//.*archive.ubuntu.com|//mirrors.ustc.edu.cn|g' /etc/apt/sources.list

    搜索软件

    您可以使用 apt search <content> 命令在软件仓库内搜索所需的软件，下面是搜索 firefox 的一个返回结果：

    $ apt search firefox
Sorting... Done
Full Text Search... Done 
(Output omitted) 

firefox/bionic-updates,bionic-security,now 72.0.2+build1-0ubuntu0.18.04.1 amd64
    Safe and easy web browser from Mozilla 

(Output omitted)

    有效输出的含义如下：

    [仓库中的包名]/[包含该软件的仓库源] [包版本] [包架构]
    [软件仓库对该包的描述]

    安装软件

    有 App Store 就直接用图形化界面装吧 orz

    您可以使用 pat install <packageName> 命令安装指定软件（权限不够的话试试 sudo）

    • 手动安装 deb 二进制包

      您可以通过 apt install path/to/file.deb 来安装手动下载的二进制 deb 文件

      • 这一操作实际上是通过 deb 格式包管理器 dpkg 完成的

      • 直接通过 dpkg -i 安装 deb 文件 不会安装相关依赖，只会报依赖缺失（坏耶）

        您可以尝试通过 sudo apt -f install 尝试抢救一下依赖缺失的问题

    • 安装预编译可执行文件

      一般流程：下载 -> 解压 -> 在 /bin 下找可执行文件 -> 开用

    • 使用源代码编译安装

      不会，大人 请看

    更新软件列表 / 本体

    计算机会在 本地 维护一个包列表，包含软件及其依赖关系信息。在执行 apt install 时会自动 递归 的安装所有依赖（所以实际上安装了一连串的依赖）。

    • 更新软件列表

      使用 apt update 命令将对本地列表进行更新 => 获取新的 软件版本 / 依赖关系 信息

      因为 apt 可能配置了若干个软件源，本命令将依次更新所有软件源并逐行输出结果

      最后的输出将指出系统中 可以被更新 的软件包总量

    • 更新软件

      请在更新软件列表后执行本命令

      使用 apt upgrade 将对比已安装软件的当前版本与软件列表中的最新版本以进行更新

      => 会自动解决新的依赖关系

    2 文件与目录操作

    查看文件内容

    cat

    您可以用以下命令在命令行中输出文件的所有内容（没有高亮捏）：

    cat [options] path/to/file

    cat 实际上被用于连接多个文件并输出，您可以使用 cat file1 file2 查看两个文件连接后的内容

    less

    区别于一次性将文件的所有内容输出至终端的 cat 命令，less 支持提供了一个可交互窗口以前后翻页显示文件与搜索等功能：

    less path/to/file

    下面是一些常用快捷键：

    按键 效果
    d / u 向下/上滚动半页
    f / b 向下/上滚动一整页
    g / G 跳转到文件开头/结尾
    j / k 向下/上移动一行
    /[pattern] 在文件中搜索 pattern
    n / N 跳转到下一个/上一个找到的 PATTERN
    q 退出

    复制文件 / 目录 cp

    # 文件复制
cp [options] srcFile dstFile

# 目录复制
cp [options] [srcFileList / srcDir] dstDir

    常用 options 如下：

    选项 备注
    -r / -R 递归复制（用于目录）
    -f 覆盖目标地址下的同名文件
    -u 同名文件较目标文件新时覆盖
    -l 创建硬链接
    -s 创建软链接

    下面是一些 sample：

    1. 在原目录下复制 file1.txt 并将副本命名为 file2.txt

      cp file1.txt file2.txt

    2. 将 file1,file2 复制至同目录下的 file 路径下

      cp file1.txt file2.txt ./file/

    3. 将 /dir1 及其子目录下的所有文件复制至同目录下的 test 路径下

      cp -r dir1 ./test/
    硬链接 vs 软链接

    • 硬链接

      与源文件指向磁盘中的同一个位置，删除一个 另一个不受影响

    • 软链接

      保存了源文件的 路径（在访问时进行替换），删除源文件后 失效

    移动文件 / 目录 mv

    类似于剪切操作，也可以用于文件重命名（大雾）

    # 文件剪切
mv [options] srcFile dstFile

# 目录剪切
mv [options] [srcFileList / srcDir] dstDir

    常用的选项有 -f / -u（含义见 cp 中的相关条目）

    删除文件 / 目录 rm

    rm [options] fileList # 使用空格连接
rm [options] dir      # 需要使用 -r 选项递归删除

    常用选项如下：

    选项 备注
    -f 忽略不存在 / 没有权限的文件和路径
    -r / -R 递归删除子路径及其下文件
    -d 删除空路径

    创建文件 / 路径

    mkdir

    mkdir [options] [dirNameList ...]

    常用的选项是 -p，会在中间目录不存在时自动创建。下面是两个常用案例：

    1. 在当前路径下创建两个路径 test1 & test2

      mkdir test1 test2

    2. 创建路径 test1/test2/test3（自动创建中间路径）

      mkdir -p test1/test2/test3

    touch

    We all like touching grass !

    touch [fileNameList]

    touch 工具本质上修改了文件的 访问&创建 时间（只是在文件不存在时会自动创建一个新的文件）

    搜索文件 / 路径 find

    # 在指定路径下寻找符合 exp 的文件
find [options] path [expression]

    常用 expression 如下表所示：

    表达式 备注
    -name '*.txt' 文件名为 *.txt
    -type d 类型为目录，-type f 表示类型为文件
    -size +1M 大小大于 1M，-1M 表示小于 1M
    -or 或运算符（用于连接两个条件）

    下面是一些常用的例子：

    1. 在 当前目录下 搜索名为 report.pdf 的文件

      find . -name `report.pdf`

    2. 全盘 搜索大于 1G 的文件

      find / -size +1G

    3. 在 用户目录下 搜索名为 node_modules 的目录

      find ~/ -name 'node_modules' -type d # 所有条件默认使用 and 连接

    解压/解压 文件 tar

    tar 来源于 Tape Archive，只是将文件本体及其 meta 信息（权限、时间戳等）打包在一起，并没有进行压缩 ！

    为了得到体积更小的文件，我们需要再对 tar 文件进行压缩：

    • 使用 gzip 算法压缩得到的文件后缀为 .tar.gz ，需要使用 -z 选项进行解压
    • 使用 xz 算法压缩后得到的文件后缀为 .tar.xz，需要使用 --xz 选项进行解压

    压缩

    我们可以使用自带的 gzip / bzip2 算法对文件进行打包压缩：

    tar [options] fileName

    常用选项如下：

    选项 备注
    -A 将一个 压缩包 的内容追加到另一个后
    -r / --append 将 一些 文件追加到一个 压缩包 中
    -c / --create 从 一些 文件创建一个 压缩包
    -t / --list 列出一个 压缩包 中的内容
    -x / --extract 解压指定压缩包
    -f / --file=ARCHIVE 使用指定的压缩包
    -C / --directory=DIR 指定输出路径

    以下选项可以指定使用的压缩算法：

    选项 备注 ｜
    -z / --gzip / --gunzip / --unzip 使用 gzip 算法 压缩 / 解压
    -j / --bzip2 使用 bzip2 算法处理文件
    -J / --xz 使用 xz 算法处理文件

    下面是一些应用实例：

    • 将 file1~3 打包为 target.tar （还没压缩捏）

      tar -c -f target.tar file1 file2 file3

    • 将 target.tar 中的文件提取到 test 路径下

      tar -x -f target.tar -C test/

    • 将 file1~3 打包，并用 gzip 算法压缩 => 得到 target.tar.gz

      tar -cz -f target.tar.gz file1 file2 file3

    • 将 target.tar.gz 解压至 test 路径下

      tar -xz -f target.tar.gz -C test/

    • 将 archive1~3.tar 中的内容追加到 archive.tar 中

      tar -Af archive.tar archive1.tar archive2.tar archive3.tar

    • 列出 target.tar 中的内容

      tar -t -f target.tar => 可以使用 tar -tv -f target.tar 打印详细信息
