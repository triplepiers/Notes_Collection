本人的 Linux 101 阅读笔记

1 包管理系统 apt

apt 的全称是 Advance Package Tool，是一个用于在 Debian、Ubuntu 或者其他 Linux 发行版上安装和移除软件的自由软件。

软件源

apt 软件源列表位于 /etc/apt/sources.list 下，您可以通过以下命令将其列出：

cat /etc/apt/sources.list | grep -v "#"

每一条返回记录将遵循以下格式：

[Archive Type] [Repository URL] [Distribution] [Components]

您也可以通过以下命令将官方源 http://archive.ubuntu.com/ 替换为国内镜像以提高速度：

sudo sed -i 's|//.*archive.ubuntu.com|//mirrors.ustc.edu.cn|g' /etc/apt/sources.list

搜索软件

您可以使用 apt search <content> 命令在软件仓库内搜索所需的软件，下面是搜索 firefox 的一个返回结果：

$ apt search firefox
Sorting... Done
Full Text Search... Done 
(Output omitted) 

firefox/bionic-updates,bionic-security,now 72.0.2+build1-0ubuntu0.18.04.1 amd64
    Safe and easy web browser from Mozilla 

(Output omitted)

有效输出的含义如下：

[仓库中的包名]/[包含该软件的仓库源] [包版本] [包架构]
    [软件仓库对该包的描述]

安装软件

有 App Store 就直接用图形化界面装吧 orz

您可以使用 pat install <packageName> 命令安装指定软件（权限不够的话试试 sudo）

  • 手动安装 deb 二进制包

    您可以通过 apt install path/to/file.deb 来安装手动下载的二进制 deb 文件

    • 这一操作实际上是通过 deb 格式包管理器 dpkg 完成的

    • 直接通过 dpkg -i 安装 deb 文件 不会安装相关依赖，只会报依赖缺失（坏耶）

      您可以尝试通过 sudo apt -f install 尝试抢救一下依赖缺失的问题

  • 安装预编译可执行文件

    一般流程：下载 -> 解压 -> 在 /bin 下找可执行文件 -> 开用

  • 使用源代码编译安装

    不会，大人 请看

更新软件列表 / 本体

计算机会在 本地 维护一个包列表，包含软件及其依赖关系信息。在执行 apt install 时会自动 递归 的安装所有依赖（所以实际上安装了一连串的依赖）。

  • 更新软件列表

    使用 apt update 命令将对本地列表进行更新 => 获取新的 软件版本 / 依赖关系 信息

    因为 apt 可能配置了若干个软件源，本命令将依次更新所有软件源并逐行输出结果

    最后的输出将指出系统中 可以被更新 的软件包总量

  • 更新软件

    请在更新软件列表后执行本命令

    使用 apt upgrade 将对比已安装软件的当前版本与软件列表中的最新版本以进行更新

    => 会自动解决新的依赖关系

2 文件与目录操作

文件权限

Linux 中的 文件 & 目录 都具有自己的权限，可以使用 ls -l 命令列出（第一列）：

$ ls -l
total 8
-rwxrw-r-- 1 ustc ustc   40 Feb  3 22:37 a_file
drwxrwxr-x 2 ustc ustc 4096 Feb  3 22:38 a_folder

  • 10 位字符串代表的信息如下：

    位数 含义
    1 文件类型
    2-4 owner 权限
    5-7 owner 同组用户权限
    8-10 其余用户权限

    其中 r 表示可读，w 表示可写，x 表示可执行；- 表示没有对应权限。

  • 权限可以用 chmod (Change file Mode bits) 修改

  • owner 可以用 chown (change file owner) 修改。

第三、四列分别表示 owner 和 owner 所属的用户组

文件系统层次结构

对于 Unix 文件系统来说，整个系统从根目录 / 开始，其他分区（C盘 / D盘）以 mount 形式挂载在这棵树上：

根目录下的各目录含义由 FHS (文件系统层次标准 Filesystem Hierarchy Standard) 定义，您可以在 官网 查看具体内容：

  • /bin 必须的程序文件，所有用户 可用
  • /boot 启动系统需要的文件

  • /dev 设备文件

    Linux 哲学认为「一切皆文件」，程序和用户可以通过读写文件的形式向设备输入/取出数据，相关细节会由对应的驱动程序处理。

  • /etc 系统和程序的配置文件

  • /home 用户的家目录，存储用户各自的文件
  • /lib 系统运行所需的库文件
  • /media or /mnt 用于挂载其他文件系统
    • /media 用于挂载光盘等可移除文件系统
    • /mnt 用于临时使用

查看文件内容

cat

您可以用以下命令在命令行中输出文件的所有内容（没有高亮捏）：

cat [options] path/to/file

cat 实际上被用于连接多个文件并输出，您可以使用 cat file1 file2 查看两个文件连接后的内容

less

区别于一次性将文件的所有内容输出至终端的 cat 命令，less 支持提供了一个可交互窗口以前后翻页显示文件与搜索等功能：

less path/to/file

下面是一些常用快捷键：

按键 效果
d / u 向下/上滚动半页
f / b 向下/上滚动一整页
g / G 跳转到文件开头/结尾
j / k 向下/上移动一行
/[pattern] 在文件中搜索 pattern
n / N 跳转到下一个/上一个找到的 PATTERN
q 退出

复制文件 / 目录 cp

# 文件复制
cp [options] srcFile dstFile

# 目录复制
cp [options] [srcFileList / srcDir] dstDir

常用 options 如下：

选项 备注
-r / -R 递归复制（用于目录）
-f 覆盖目标地址下的同名文件
-u 同名文件较目标文件新时覆盖
-l 创建硬链接
-s 创建软链接

下面是一些 sample：

  1. 在原目录下复制 file1.txt 并将副本命名为 file2.txt

    cp file1.txt file2.txt

  2. 将 file1,file2 复制至同目录下的 file 路径下

    cp file1.txt file2.txt ./file/

  3. 将 /dir1 及其子目录下的所有文件复制至同目录下的 test 路径下

    cp -r dir1 ./test/
硬链接 vs 软链接

  • 硬链接

    与源文件指向磁盘中的同一个位置，删除一个 另一个不受影响

  • 软链接

    保存了源文件的 路径（在访问时进行替换），删除源文件后 失效

移动文件 / 目录 mv

类似于剪切操作，也可以用于文件重命名（大雾）

# 文件剪切
mv [options] srcFile dstFile

# 目录剪切
mv [options] [srcFileList / srcDir] dstDir

常用的选项有 -f / -u（含义见 cp 中的相关条目）

删除文件 / 目录 rm

rm [options] fileList # 使用空格连接
rm [options] dir      # 需要使用 -r 选项递归删除

常用选项如下：

选项 备注
-f 忽略不存在 / 没有权限的文件和路径
-r / -R 递归删除子路径及其下文件
-d 删除空路径

创建文件 / 路径

mkdir

mkdir [options] [dirNameList ...]

常用的选项是 -p，会在中间目录不存在时自动创建。下面是两个常用案例：

  1. 在当前路径下创建两个路径 test1 & test2

    mkdir test1 test2

  2. 创建路径 test1/test2/test3（自动创建中间路径）

    mkdir -p test1/test2/test3

touch

We all like touching grass !

touch [fileNameList]

touch 工具本质上修改了文件的 访问&创建 时间（只是在文件不存在时会自动创建一个新的文件）

搜索文件 / 路径 find

# 在指定路径下寻找符合 exp 的文件
find [options] path [expression]

常用 expression 如下表所示：

表达式 备注
-name '*.txt' 文件名为 *.txt
-type d 类型为目录，-type f 表示类型为文件
-size +1M 大小大于 1M，-1M 表示小于 1M
-or 或运算符（用于连接两个条件）

下面是一些常用的例子：

  1. 在 当前目录下 搜索名为 report.pdf 的文件

    find . -name `report.pdf`

  2. 全盘 搜索大于 1G 的文件

    find / -size +1G

  3. 在 用户目录下 搜索名为 node_modules 的目录

    find ~/ -name 'node_modules' -type d # 所有条件默认使用 and 连接

解压/解压 文件 tar

tar 来源于 Tape Archive，只是将文件本体及其 meta 信息（权限、时间戳等）打包在一起，并没有进行压缩 ！

为了得到体积更小的文件，我们需要再对 tar 文件进行压缩：

  • 使用 gzip 算法压缩得到的文件后缀为 .tar.gz ，需要使用 -z 选项进行解压
  • 使用 xz 算法压缩后得到的文件后缀为 .tar.xz，需要使用 --xz 选项进行解压

压缩

我们可以使用自带的 gzip / bzip2 算法对文件进行打包压缩：

tar [options] fileName

常用选项如下：

选项 备注
-A 将一个 压缩包 的内容追加到另一个后
-r / --append 将 一些 文件追加到一个 压缩包 中
-c / --create 从 一些 文件创建一个 压缩包
-t / --list 列出一个 压缩包 中的内容
-x / --extract 解压指定压缩包
-f / --file=ARCHIVE 使用指定的压缩包
-C / --directory=DIR 指定输出路径

以下选项可以指定使用的压缩算法：

选项 备注 ｜
-z / --gzip / --gunzip / --unzip 使用 gzip 算法 压缩 / 解压
-j / --bzip2 使用 bzip2 算法处理文件
-J / --xz 使用 xz 算法处理文件

下面是一些应用实例：

  • 将 file1~3 打包为 target.tar （还没压缩捏）

    tar -c -f target.tar file1 file2 file3

  • 将 target.tar 中的文件提取到 test 路径下

    tar -x -f target.tar -C test/

  • 将 file1~3 打包，并用 gzip 算法压缩 => 得到 target.tar.gz

    tar -cz -f target.tar.gz file1 file2 file3

  • 将 target.tar.gz 解压至 test 路径下

    tar -xz -f target.tar.gz -C test/

  • 将 archive1~3.tar 中的内容追加到 archive.tar 中

    tar -Af archive.tar archive1.tar archive2.tar archive3.tar

  • 列出 target.tar 中的内容

    tar -t -f target.tar => 可以使用 tar -tv -f target.tar 打印详细信息

3 进程控制

查看本节之前，请使用 sudo apt install htop 来安装 htop，以便即时查看进程的各个属性

在 MacOS 环境下，您可以通过 HomeBrew 进行安装：brew install htop

预备知识

进程标识符 PID

  • PID 的全称是 Process Identifier，这是一个用于唯一标识进程的 数字。
  • 当用户希望 挂起 / 继续 / 终止 某一特定进程时，可以使用 PID 作为索引。
  • 在 Htop 中显示在最左侧，可以通过单击进行 升/降 序排序。

进程状态

  • 在 OS 课程中，我们粗略的将进程划分为 Running / Ready / Blocked 三类（忽略创建销毁状态）

  • 而在实际 Linux 系统中，进程状态的分类要更加复杂：

    R: running            => 运行 / 就绪 态
S: sleeping           -> 阻塞（可被中断）
D: disk sleep         -> 阻塞（不可被中断 => 在磁盘 IO 上阻塞）
T: traced / stopped   => Ctrl+Z 导致的挂起 / 被调试工具跟踪
Z: zombie             => 已经结束但仍占用一个 PID

服务

大多数 Linux 发行版的 init 方案为 systemd，对应系统服务管理命令 systemctl

  • 我们可以通过 systemctl status 命令查看列出系统中正在运行的服务
  • 也可以通过 systemctl list-units 查看所有可提供的服务内容

基本操作

查看当前运行的进程

  • 您可以使用 htop 方便的查看当前运行的所有进程，以及 CPU / 内存占用与系统负载信息

  • 您也可以使用 ps(process status) 命令列出 本终端运行 的相关进程，或使用 ps aux 列出 所有 正在运行的进程

前后台切换

  • 在命令后加上 & 即可在后台运行程序：./a.out &，这允许我们在执行该程序的同时操作 shell 完成其他的摸鱼操作

  • 如果需要将 正在运行的程序切换至后台，我们可以通过 Ctrl + Z 发送 SIGTSTP 使该进程挂起

    执行成功后返回了 [1]  + 85366 suspended  htop，行首为其对应的任务代号

    您可以通过 jobs 指令查看当前 shell 上的所有相关进程

  • 您可以使用 bg %n / fg %n / kill %n 进行前后台切换 / 杀死进程

  • + 指代 fg/bg 命令默认操作的任务，- 表示 + 号进程结束后进行顶替的任务

    您也可以通过 %+ / %- 指代这两个特殊的任务

终止进程

  • 有多种信号 可以引发进程死亡。标准的终止信号为 SIGTERM，表示进程 自然死亡。

  • kill 命令

    • 格式为 kill [options] PID，可以使用 kill -l 查看所有信号的对应名称

    • 在不添加参数的情况下（kill PID），将默认发送 SIGTERM 作为信号参数

    • SIGKILL 表示 立即退出，可以通过 kill -9 PID 进行发送

nohup

  • 通过 SSH 进行远程连接时，终端中执行的程序会在连接断开时被关闭

    => 这是由于终端在关闭时会向其中运行的 所有进程 发送 SIGHUP (signal hangup)，其默认动作即为退出程序运行

  • 您可以通过 nohup 程序 & 屏蔽 SIGHUP 对程序的影响，下面是一个例子：

    $ nohup ping 101.ustclug.org &
[1] 19258
nohup: ignoring input and appending output to '/home/ustc/nohup.out'

    上述指令屏蔽了 SIGHUP 对 ping 的影响，并将其输出写到 nohup.out 文件中

4 用户与用户组

Linux 用户分类

您可以查看 /etc/password 文件来获取本系统中的用户信息

在 Linux 中，用户被分为三类：

  1. 根用户 root

    根用户在 Linux 系统中拥有最高权限，可以执行 任何操作（比如删除系统文件）

    其相关用户数据存储在 /root 路径下

    当我们使用 sudo 并通过密码验证后，便会以 根用户 的身份执行命令（由于 apt 安装的软件存储在系统目录下，所以必须使用 sudo 进行安装）

    您可以通过 sudo !! 执行上一条命令（避免重新敲一遍）

    谨慎使用 root 权限执行命令，特别是 rm -rf / !

  2. 系统用户

    是由系统 / 相关程序创建的（比如 mysql），用于执行服务等系统任务

    为了区别于普通用户，系统用户的 UID 会处于一个指定范围中，并默认禁止使用密码登录

    为了避免系统寄掉，请 不要随便删除这些用户

  3. 普通用户

    可以登录系统，并对 自己 home 目录 下的文件进行操作（一般为 /home/username）

    无法直接修改系统配置，也不能为 系统环境 安装 / 卸载软件

    • 您可以通过 sudo -u username 命令 来以其他用户的身份执行命令（除 root 外）
    • 您也可以通过 su username 切换为其他用户（切换至根用户需要用 sudo su）

用户组

就是若干用户构成的集合，方便为一批用户设置权限。

  • 你可以通过 groups 命令查看自己所属的用户组。

    一般会有若干个，包含一个与自己同名的用户组（在创建用户时自动创建）

  • 普通用户会在配置部分软件时使用用户组机制。

    比如在配置 Docker 时，可以将自己加入 docker 用户组，从而不需要 root 权限也能访问对应接口。

  • 用户组和用户一样具有编号 GID

用户配置

  • 修改密码

    passwd UserName # 缺省用户名时，修改当前用户密码

  • adduser 命令（Debian 专属）

    • 添加用户 sudo adduser userName
    • 添加用户组 sudo adduser --group groupName
    • 将用户添加至指定用户组 sudo adduser userName groupName

5 shell 脚本

常见概念

别名 alias

bash 允许使用使用别名代指制定命令，例如：

alias la = `ls -a`

使得您可以通过 la 来执行长命令 ls -a，从而迅速完成复杂操作。

您可以通过 alias 命令查看当前系统中存在的所有别名。

脚本运行

运行方式

以下是几种运行 bash 脚本的方法：

  • 在 指定的 shell 下运行（脚本程序文件名将作为 shell 的 第一个参数）

    bash show.sh [options]

  • 使用 . 命令执行：

    . ./show.sh [options]

  • 将脚本设置为 可执行，随后像外部命令一样执行：

    chmod a+x show.sh # 增加 x 权限
./show.sh [options]

许多 bash 脚本会在文件首行写上 #!/bin/bash

当文件首行存在 sehbang #! 且以 可执行模式 执行，则执行 ./foo.sh 等价于执行 /bin/bash ./foo.sh（指定了解释器）

多命令同行

Bash 也支持在同一个行中安排多个命令：

分隔符 说明
; 顺序执行
&& 先执行前面的命令，若成功，才接着执行后面命令；若失败，不执行后面命令
\| 先执行前面的命令，若成功，不执行后面命令；若失败，才执行后面命令
后缀 & 后台执行
组命令

  • { 命令1; 命令2; … }，组命令在 shell 内执行，不会产生新的进程，注意花括号和命令之间的空格。

  • (命令1; 命令2; …)，组命令会建立独立的 shell 子进程来执行组命令，这里的圆括号周围并不需要空格。

# 当前在家目录
(cd /tmp; pwd;)   # 仍然在家目录
{ cd /tmp; pwd; } # 移动到了 /tmp 目录

shell 变量

  • shell 不支持数据类型，所有的变量值均被视为 字符串

  • 从 赋值形式 来看，shell 变量可被分为四类：用户自定义、环境变量、位置变量和预定义特殊变量。

用户自定义变量

使用命令 name=串 定义

  • 等号两侧不能有空格，串内包含空格时需要用双引号包裹
  • 可以通过 $name / ${name} 使用变量
  • 可以通过 unset name 删除已定义变量
  • 使用 未定义变量 将以默认值 空串 处理
环境变量

  • 用户可以通过 env 命令列出所有已定义的环境变量。

  • 当用户登录系统时，Linux 都会为其创建一个由一组环境变量定义的默认工作环境。下面是一些常用的环境变量：

    • HOME 当前用户的主目录，一般为 /home/usrname
    • PATH 命令搜索路径（以冒号分隔）。输入命令名时，系统会以此搜索 PATH 给出的路径，以确定对应程序是否存在
    • PWD 当前工作路径
    • SHELL 默认 shell 的路径

  • 可以使用 export name=val 命令定义环境变量（在 退出前 有效）

位置变量

  • Shell 解释用户的命令时，把命令程序名后面的所有字串作为程序的参数。分别对应 $1、$2、$3、……、$9，程序名本身对应 $0。

  • 可用 shift <n> 命令，丢弃开头的 n 个位置变量，改变 $1、$2、$3 等的对应关系。

  • 可用 set 命令，重置整个位置变量列表，从而给 $1、$2、$3 等赋值。

下面是一个用例：

set one two three
echo $1 $2 $3
# 输出 one two three
shift 2
echo $1 $2 $3
# 输出 three => 此时 $2 和 $3 已不存在
特殊变量

用户 只能引用 以下变量，不能对其进行直接改变 / 重置：

变量 说明
$# 命令行上参数个数（不包括 $0）
$? 最后命令的退出代码：0 表示成功，其它值表示失败
$$ 当前进程 PID
$! 最近一个后台运行进程的进程号
$* 命令行所有参数构成的一个字符串
$@ 用双引号括起的命令行各参数拼接构成的一个字符串
特殊字符

  • 反斜杠

    消除单个字符的特殊含义。

    • 包含空白字符（空格和制表符）、反斜杠本身、各种引号，以及 $、! 等。
    • 与其他语言不同，shell 中反斜杠不会将普通字符转义为其他含义（例如 \n 不会被视作换行符）。

  • 双引号

    消除空白字符切分参数的特殊含义，但是很多其他特殊字符的特殊含义仍然保留。

  • 单引号

    能消除所有特殊字符的特殊含义（包括反斜杠）

    因此单引号字符串中不能使用反斜杠转义单引号本身

  • 反引号（`）括起的字符串被 shell 解释为命令，执行时用命令输出结果代替整个反引号对界限部分。

    • 与反引号相同的语法是 $(command)，它的好处是界限更明确，且可以嵌套。
输入输出

  • 我们可以使用 read 命令获取用户输入，并将其赋给指定变量

    read -p "Enter your name: " name # -p 后为输入提示，结果赋给变量 $name
echo $name

    \ 将被默认识别为转义符号（\( => (），可以添加 -r 参数进行 escape

    read -r -p "Enter your message: " message

  • 输出可以通过 echo 或 print 命令实现

    • echo

      -n 表示取消输出行末的换行符，-e 会解析文本中的转义字符（如 \n）

      $ echo "Hello $name.\nWelcome to bash!"    # \n 没有被转义成换行 
Hello linux.\nWelcome to bash! 

$ echo -e "Hello $name.\nWelcome to bash!" # 添加 -e 选项转义字符
Hello linux. 
Welcome to bash!
算数运算

  • bash 支持 计算表达式 / 使用let命令对变量进行赋值

    • 表达式的运算符两侧与操作数之间 必须有空格，圆括号需要转义

    • let 运算符两侧可以没有空格，括号无需转义，但 等号两侧不能有空格

      $ let a=0 
$ echo $a 
0 
$ let a++ 
$ echo $a
1

  • 支持五种种基本运算 +- and \*(转义乘法)/ and %

  • 您也可以使用 (( expr )) 语法进行计算 / 赋值
    • 这是一条完整的命令，命令返回值为 0/1
    • 当且仅当运算结果为零时返回1，其余情况返回0 
      $ (( 1 + 1 ))
$ echo $?  
0 # 结果为 2，所以返回值为 0
$ (( 1 - 1 ))
$ echo $?  
1 # 结果为 2，所以返回值为 0

# 使用 $((expr)) 可以将计算结果作为命令的一部分
$ let a=0
$ echo $((a+=a/a))
1
条件表达式

条件表达式有以下两种书写形式：

  1. test 条件表达式

  2. [ 条件表达式 ]，表达式与方括号之间应有空格

  3. 字符串

    • 相等 / 不等比较 str1 = str2 / str1 != str2
    • 字符串长度（返回 True / False）
      • str / -n str 在 len > 0 时返回真
      • -z str 仅在空串（len = 0）时返回真

  4. 数值比较

    基本格式为 num1 [option] num2，支持以下选项：

    • 相等 -eq / 不等 -ne
    • 大于 -gt / 大于等于 -ge
    • 小于 -lt / 小于等于 -le

  5. 文件状态

    基本格式为 [option] fileName，支持以下选项：

    • -r 存在且可读
    • -w 存在且可写
    • -x 存在且可执行
    • -f 存在且为文件，-d 存在且为目录
    • -s 存在且内容非空

  6. 复合逻辑

    • 否运算 ! expr
    • 与运算 expr1 -a expr2
    • 或运算 expr1 -o expr2
流程控制

bash 中的 if 支持多分支与嵌套，以 fi 表示结构结束：

if cond1
then
    branch1
elif cond2
then
    branch2
else
    branch3
fi

switch-case 中的每个 case 以 ) 收尾，多个离散值以 | 分隔，esac 表示结构结束：

case <var> in
val1 | val2)
    branch1
    ;;
val3)
    branch2
    ;;
*) # default
    branch3
    ;;
esac

支持 for 循环与 while / until 循环，均以 done 表示结构结束：

for var in list
do
    commands $var
done

while cond
do
    branch
done

until cond
do
    branch
done

同时支持 break [n] / continue [n] ，其中 n 表示跳出/跳过 N 层循环

函数

在 bash 中定义函数的语法如下：

# 先定义后使用
function funcName {
    [commandsList]
    [return <int>]
}

funcName arg1 arg2 ... # 函数调用
  • 函数参数可以通过 $n 来获取
  • 返回值可以通过 return 显式返回 / 以最后一条命令运行结果作为返回值
  • return 仅结束 函数，exit 结束 当前脚本

I/O 重定向

  • 默认情况下命令从 stdin 读取出入，并将结果输出至 stdout（两者默认情况下都是终端）

  • 使用重定向可以使命令 从文件中读取输入 / 输出结果至文件。

    • 下面是对 echo 命令进行输出重定向的例子：

      # 输出至文件 out_file（覆盖旧内容）
echo "Hello World" > out_file
# 将输出 追加 至文件 out_file 末尾
echo date >> out_file
# > & >> 均会在文件不存在时创建文件

    • 下面是输入重定向的案例：

      command < in_file            # 仅输入重定向
command < in_file > out_file # 输入输出均重定向

管道

  • pipe 的操作符为 |，用于将左侧命令的 stdout 作为右侧命令的 stdin
  • pipe 不会处理 stderr

下面是一个例子（列出所有包含 bin 字符串的 文件/目录 名）：

ls / | grep bin

网络下载

Wget

支持通过 HTTP 或 FTP 协议从因特网中检索并下载文件

  • 支持以非交互方式工作 => 在用户注销后 继续在后台工作
  • 在不稳定连接条件下支持断点续传
  • 支持 HTML 页面 & FTP 站点的 递归检索，可用于获取网站镜像 / 类似爬虫一般遍历网络
  • 可在获取文件时增加时间戳，自动识别远程文件是否更改 / 检索最新版本
  • 支持代理服务器

基本命令结构为 wget [options] [url]，可以通过 wget -h 获取帮助。常见选项如下：

选项 备注
-i / --input-file=File 从本地/外部文件中获取 URL
-O / --output-document=File 将输出写入指定文件
-b / --background 后台运行
-d / --debug 打印调试信息

您可以通过下列命令 批量下载 文件 list.txt 中给出的连接：

wget -i list.txt

cURL

  • 是一个利用 URL 语法在命令行下工作的文件传输工具。支持自定义各种请求参数，可用于 模拟 Web 请求。

  • 可以通过 curl -h 获取帮助，下面是一些常用选项

    选项 备注 ｜
    -o 将下载数据输出至指定文件（需要指定文件名）
    -O 将下载数据保存至文件（使用URL中默认的文件名）
    -I 只显示 response header

  • 下面是一些使用案例：

    • 在命令行输出 Bing 主页的代码 

      curl "https://cn.bing.com"

    • 将 Bing 主页代码输出至文件 bing.html 

      # 输出重定向
curl "https://cn.bing.com" > bing.html
# 也可以使用 -o 指定输出文件名
curl -o bing.html "http://cn.bing.com"

    • 下载 USTCLUG 的 logo

      curl -O "https://ftp.lug.ustc.edu.cn/misc/logo.png"

文本处理

文本统计 wc

wc 是一个常用的文本统计工具，可以用于输出文本的行数、单词数与字节数：

wc fileName
427 2768 20131 fileName # 依次对应 行数，单词书，字节数
  • 在存在中文文本时，应使用 wc -m 命令考虑宽字符
  • 由于中文文本的分词算法问题，wc 无法准确统计中文文本的单词数量

文本比较 diff

diff 用于比较两个文件的不同：

  • 命令将列出 文本 文件的详细差异，对 二进制 文件只会说明是否存在差异
  • 可以使用 -w 选项忽略空白字符，使用 -b 忽略空白字符数量变化

基本命令格式为 diff fileName1 fileName2

开头与结尾 head & tail

用于显示文件 开头 / 结尾 处制定数量的字符：

  • 不添加参数时，默认显示 前/末 10 行的内容
  • 可以使用 -n <num> / -<num> 指定需要显示的 行数
  • 可以使用 -c <num> 制定需要显示的 字节数
  • 对于 tail 命令，可以使用 -f 显示文件末尾更新的内容

下面是一些使用案例： 

head file        # 显示前十行
head -25 file    # 显示前二十五行
head -c 10 file  # 显示前10个字符
tail -n 10 file  # 显示末十行
tail -f cata.log # 显示 log 文件的更新内容

查找与替换

grep 命令用于在文本中查找制定的字符串，下面是一些用例：

grep 'hello' file # 查找 file 中包含 'hello' 的行
ls | grep 'file'  # 查找 当前路径下 名字包含 'file' 的文件
grep -i 'Systemd' file # 查找包含 'Systemd' 的行（忽略大小写）
grep -R 'hello' . # （递归）查找当前目录下 内容 包含 'hello' 的文件

sed 命令用于替换文本中的制定字符串，下面是一些用例：

sed 's/hello/world/g' file  
# 将文件 file 中的 hello 全局（global）替换为 world 后输出
sed 's/hello/world/' file  
# 将文件 file 的每一行 第一个 出现的 hello 替换为 world 后输出
echo 'helloworld' | sed 's/hello/world/g'  # 管道也是可以的
sed -i 's/hello/world/g' file      # -i 参数会直接写入文件，操作前记得备份哦！
sed -i.bak 's/hello/world/g' file  # 当然，也可以让 sed 帮你备份到 file.bak

最常见的应用场景是替换软件源：

sudo sed -i 's/cn.archive.ubuntu.com/mirrors.ustc.edu.cn/g' /etc/apt/sources.list
