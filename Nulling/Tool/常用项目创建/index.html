    常用项目创建

    Venv

    1. 创建虚拟环境 python -m venv [name]
    2. 激活虚拟环境 source ./bin/activate

    3. 关闭虚拟环境 deactivate

    4. 读取列表并安装 pip3 install -r requirements.txt

      奇妙报错可以尝试通过 pip install --upgrade pip setuptools==45.2.0 解决

    5. 从 virtualenv 导出依赖列表 pip freeze >requirements.txt

    Jupyter Notebook

    1. 安装 pip3 install jupyter
    2. 解决 command not found
      1. 查看安装路径 pipe show jupyter
      2. 添加 PATH 
        vi ~/.zshrc
# 添加以下内容（取到 Python 版本号为止的路径）
export PATH="/Users/shen/Library/Python/3.9/bin:$PATH"
# 重载命令
source ~/.zshrc
    3. 启动服务 
      # 默认端口（8888）
jupyter notebook
# 指定端口
jupyter notebook --port 8080

    4. 删除单元格

      选中单元格使其呈蓝色，输入 dd

    Anaconda

    您可以通过 brew install anaconda 命令以使用 homebrew 进行安装

    以下指令适用于 >= 23.7.4 版本

    • 检查当前安装的 Anaconda 版本 conda --version
    • 目前安装的所有环境 conda info --envs
    • 检查可安装的 Python 版本 conda search --full-name python
    • 创建环境 conda create --name [envName] python=[PyVersion]，删除环境 conda remove -n [envName] --all
    • 激活指定环境 source activate [envName]，退出当前环境 conda deactivate

    静态站点成器

    Mkdocs

    • 安装与启动
      1. 创建 python3 虚拟环境
      2. 使用 pip 安装 mkdocs - pip install mkdocs
      3. 使用 mkdocs --version 检验是否正确安装
      4. 创建项目 mkdocs new prj_name
      5. 开启服务 mkdocs serve

    • 部署到仓库的 gh-pages 分支

      mkdocs gh-deploy

    • 使用 material 主题

      • 安装 pip install mkdocs-material
      • 配置 theme: name: material

    VuePress

    用 VuePress 搭的 Wiki 可以拿 OverGrad 组的 Best Wiki 提名属实是有点抽象了

    这里是 官方文档

    • 安装

      1. 使用包管理器进行初始化：npm init
      2. 将 VuePress 安装为本地依赖：npm install -D vuepress

      3. 在 /docs 路径下创建第一篇文档：mkdir docs && cd docs && touch README.md

        • VuePress 的所有路径都是相对于 /docs 而言的
        • README.md 可以通过每一级路由的默认路径进行访问

      4. 在 package.json 中添加下列脚本（可选）

        {
    "scripts": {
    "docs:dev": "vuepress dev docs",
    "docs:build": "vuepress build docs"
    }
}

      5. 启动项目：npm run docs:dev

        这将在 8080 端口启动一个热重载的开发服务器

    CSS 兼容性处理

    配置完成后，您就可以使用 npm run build 命令一键生成 “兼容旧浏览器 · minimized” 的独立 CSS 文件了

    • 环境搭建

      # 跳过向导生成 npm 项目
npm init -y
# 局部安装 webpack5
npm install webpack webpack-cli path

    • 安装开发依赖 postcss

      npm install postcss-loader postcss-preset-env css-loader autoprefixer mini-css-extract-plugin -D

    • 配置 webpack(5)

      webpack.config.js
      // 导入需要的插件
const path = require('path');
const MiniCssExtractPlugin = require('mini-css-extract-plugin');
const CssMinimizerPlugin = require('css-minimizer-webpack-plugin');
// 设置 node.js 环境变量为 开发环境
process.env.NODE_ENV = "development";
// 配置
module.exports = {
    resolve: {
        extensions: ['.js', '.json', '.wasm'],
        alias: {
            '@': path.resolve(__dirname,'src' )
        }
    },
    module: {
        rules: [
            {   // 处理 css 文件
                test: /\.css$/,
                use: [
                    // 成为单独的 css 文件（不是作为 <style> 标签
                    MiniCssExtractPlugin.loader, 
                    'css-loader',
                    // 配置兼容选项（配置在 postcss-config.js中）
                    {
                        loader: 'postcss-loader'
                    }
                ]
            }
        ]
    },
    plugins: [
        new MiniCssExtractPlugin({   // 使用 extract 插件
            filename: './built.css'  // 统合输出的文件名
        })
    ],
    optimization: {
        minimize: true,
        minimizer: [
            new CssMinimizerPlugin({ // 使用 minimize 插件
                test: /\.css$/g
            })
        ]
    },
    mode: "development"
}

    • 配置 package.json

      package.json
      {
    "scripts": {
        ...,
        "build": "webpack" // 使用 npm run build 即可打包
    },
    "browserslist": {      // 生产/开发模式下兼容的浏览器信息
        "development": [
            "last 20 chrome version",
            "last 20 firefox version",
            "last 20 safari version"
        ],
        "production": [
            ">0.2",
            "not dead",
            "not po_mini all"
        ]
    }
}

    • 配置 postcss-config.js

      postcss-config.js
      const autoprefixer = require('autoprefixer');
module.exports = {
    plugins: () => {
        require('postcss-loader-env')();
        autoprefixer({
            browsers: [
            "last 20 chrome version",
            "last 20 firefox version",
            "Safari >= 10",
            "ie > 8"
            ]
        })
    }
}

    MySQL

    • 命令行登录 mysql -u root -p

    • 获取当前日期并规范化

      • 获取 YYMNMDD 格式的今天日期：select date_format(now(), '%y%m%d');

      • 获取明天的日期：select date_sub(curdate(), interval -1 day);

      • 套娃：获取明天日期并格式化

        select date_format(date_sub(curdate(),interval -1 day), '%y%m%d');

    创建定时事件

    开启事件计划 -> 定义待执行的函数 -> 设置执行的 interval

    1. 检查 MySQL 是否开启了事件计划

      show variable like 'event_scheduler'

      如果显示对应变量为 OFF 则需要手动设置（命令行中的设置将在重启数据库时失效）

      set global event_scheduler=1;

    2. 创建测试用的数据表 user

      CREATE TABLE `user` (
  `user_id` int(10) unsigned NOT NULL AUTO_INCREMENT COMMENT '自增主键',
  `user_name` varchar(20) DEFAULT NULL COMMENT '用户名',
  `age` int(11) DEFAULT NULL COMMENT '年龄',
  `create_time` datetime DEFAULT NULL COMMENT '创建时间',
  PRIMARY KEY (`user_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8;

    3. 创建存储过程 user_procedure -> 等待定时调用的函数

      可以用 select CURDATE() 查看 YYYY-MM-DD 格式的当前日期

      -- 先删除存储过程
DROP PROCEDURE IF EXISTS user_procedure; 
-- 创建存储过程
delimiter $$
create procedure user_procedure()
begin
-- now() 用于存入当前时间戳
insert into user(user_name , age , create_time) values ('张萌' , 26 , now());
end$$
-- 恢复分隔符号
delimiter ;

      可以手动调用一次过程进行验证

      call user_procedure();

    4. 创建定期调用事件

      每隔一分钟进行一次插入，从下一分钟开始执行

      -- 先删除事件
DROP EVENT IF EXISTS user_event ;
-- 创建名字为user_event的事件
CREATE EVENT `user_event`
-- 每隔一分钟执行一次，开始执行时间为当前时间的下一分钟
ON SCHEDULE EVERY 1 MINUTE STARTS DATE_ADD(now(), INTERVAL 1 MINUTE) 
ON COMPLETION NOT PRESERVE
ENABLE
DO call user_procedure()    -- 指定要执行的存储过程
;

      有关事件的相关操作语句如下：

      select * from information_schema.`EVENTS`;             -- 查看所有定时事件
alter event user_event on completion preserve enable;  -- 开启定时任务
alter event user_event on completion preserve disable; -- 关闭定时任务
