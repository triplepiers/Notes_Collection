Type Script

TS 是 JS 的超集，可以在任何支持 JS 的平台中执行（但不能直接用 JS 解析器执行）

0 Set Up

需要预装 Node.js 环境

  1. 使用如下命令全局安装 TS

    npm i -g typescript
# 可以使用 tsc --version 验证

  2. Hello World

    降智贴士：可以用 touch [fileName] 创建文件

    console.log('hello TS')

  3. 编译成 JS（任意版本）

    # 编译：在同路径下生成同前缀的 JS 文件
tsc [TS fileName]
# 运行：不生成 JS 文件直接执行
node [TS fileName]

编译选项

  • 编译目录下所有 TS 文件 tsc

  • 自动编译单个文件 -w --watch

    （命令行得一直开着）自动监视指定文件，在修改后重新编译

  • 自动编译项目

    需要在项目根路径下配置 tsconfig.json

    tsconfig.json
    {
    # 需要包含的文件
    "include": [
        "./**/*"           # 单个星号表示任意文件，两个表示任意目录
    ],
    # 需要排除的文件
    "exclude": [
        "./src/hello/**/*" # 排除 src/hello 下的所有文件
    ],
    # 合并配置文件
    "extends": "./configs/base", # 自动插入 config/base.json 下的所有内容
    # 需要编译的文件列表（有 include 就不用了）
    "files": [
        "./hello.ts"
    ],
    # 编译选项
    "compilerOptions": {
        # 目标版本
        "target": "ES3" / "ES6" / ... ,
        # 模块化方案
        "module": "ES6" / "commonjs",
        # 库（一般不指定）
        "lib": [ "dom" ],
        # 输出路径
        "outDir": "./dist"
        # 输出文件（合并为同一个文件，模块化会 bug）
        "outFile": "./dist/app.js",
        # 允许 JS 文件，设为 false 时忽略 include 中的 JS 文件
        "allowJS": true / false,
        # 是否按 TS 语法检查 JS 文件中的代码是否符合规范
        "checkJS": true / false,
        # 是否移除注释
        "removeComments": true / false,
        # 是否输出编译结果（JS 文件）=> 只是检查一下语法
        "noEmit": true / false,
        # 报错时不生成 JS 文件
        "noEmitOnError": true / false,

    # 语法检查选项
        # 总开关（后面选项全为 true）
        "strcit": true / false,
        # 编译结果是否使用严格模式
        "alwaysStrict": true / false, # 有模块就默认采用严格模式了
        # 是否将未声明类型的变量自动推导为 any
        "noImplicitAny": true / false,
        # 是否允许类型不明确的 this 指针（需要手动指定）
        "noImplicitThis": true / false,
        # 严格检查空值（DOM 是否成功获取对象）
        "strictNullChecks": true / false
    },
}

    webpack 打包

  • 生成 npm.json => 在根目录下用 npm init -y 指令进行初始化

  • 使用命令 npm i -D webpack webpack-cli typescript ts-loader 安装开发依赖

    目测 npm 比较卡，cnpm 速度非常给力

  • 编写 webpack 配置文件

    webpack.config.js
    // 引入包
const path = require('path');
// 配置信息
module.exports = {
    // 指定入口文件
    entry: "./src/index.ts",
    // 指定输出位置
    output: {
        path: path.resolve(__dirname, 'dist'),
        filename: "bundle.js" // 输出文件名
    },
    // 指定 webpack 打包时使用的 loader
    module: {
        rules: [
            {
                // 规则生效路径（所有以 ts 结尾的文件）
                test: /\.ts$/,
                // 匹配文件使用的 loader（需要安装对应依赖）
                use: 'ts-loader',
                // 需要排除的文件
                exclude: /node-modules/te
            }
        ]
    },
    // 设置可作为 module 的文件
    resolve: {
        extensions: ['.ts', '.js'] // 指定文件后缀
    }
};

    tsconfig.json
    {
    "compileOptions": {
        "module": "ES6",

  • 使用 webpack-dev-server 插件创建热更新服务器（但是只有 TS 热更新捏）

    package.json
    {
    ...,
    "scripts": {
        ...,
        "start": "webpack serve --open chrome" 
        # 执行 npm start 时，使用 chrome 作为默认浏览器
    },
    ...

  • 使用 clean-webpack-plugin 插件在每次更新前清空 dist 目录

    webpack 每次只是同名覆盖（但不会删掉无用的输出文件）

    webpack.config.js
    const { CleanWebpackPlugin } = require('clean-webpack-plugin')
module.exports = {
    plugins: [
        ...,
        new CleanWebpackPlugin()
    ]
}

  • 使用 babel 解决兼容性问题

    cnpm i -D @babel/core @bebel/preset-env babel-loader core-js
    webpack.config.js
    // 为 ts 文件添加新的 loader
rules: [
    {
        use: [
            {
                // 指定加载器
                loader: 'babel-loader',
                // babel 配置项
                options: {
                    // 设置目标环境
                    presets: [
                        [
                            // 环境插件
                            "@babel/preset-env",
                            // 配置信息
                            {
                                // 需要兼容的浏览器版本
                                targets: {
                                    "chrome": "88",
                                    "ie":     "11"
                                },
                                "corejs": "3", // 符合安装的版本
                                "useBuiltIns": "usage" // 按需加载
                            }
                        ]
                    ]
                }
            },
            'ts-loader'
        ]
        # loader 从后往前执行，先用 ts-loader 转新标准 JS，再用 babel 转旧的
    }
]

    1 类型

  • TS 支持通过类型声明指定变量类型，在 赋值 时自动检查并在不符合时报错

    变量名小写

    类型声明
    // 变量声明
let a: number;         // 指定变量 a 为 number 类型
let b: boolean = true; // 声明并赋值
let c = "str";         // 自动类型推断为 string

// 常量声明
let a: 10;             // 使用字面量 10 初始化为对应常量
let b: "male" | "female";
b = "male"             // 合法
b = "check"            // 非法（我觉得很抽象）

// 联合类型（更离谱了）
let c: boolean | string
c = true, c = "hello" // 均合法

  • 同时，TS 也需要对函数的返回值与形参类型进行指定，并指定形参个数

    JS 并不考虑参数的类型与个数

    function sum(a: number, b: number): number{
    return a + b;
}
// 由于兼容 JS 语法，您也可以啥也不指定
// 函数返回值也可以指定为 “联合类型” => 在不同的分支中返回不同类型值

let d;                //  == let d: any，可以赋任意类型值 / 赋值给任意类型变量

// 不知道变量类型的时候更推荐声明为 unknown
let e: unknown;       // 不能赋值给非 unknown 类型变量（
let s: string;

if (typeof e === "string") {               // 类型推断
    s = e;
}

s = e as string | s = <string>e            // 类型断言（关闭编译报错）

// object 类型
let o: object / let o: {}                  // 任意结构对象
let o: { name: string }                    // 指定结构对象必填属性（赋 {} 报错）
let o: { age?: string }                    // ? 前为可选属性
let o: { a:string, [propName: string]:any} // 除 a 外可填任意数量任意类型属性
// 具有大脑残缺美的写法（勉强算是继承？）
let o: {name:string} & {age:number}        // 和写在一个大括号里没区别

// function 类型
let f: (a: number, b: number) => number;   // 两数字参数返回数字的函数
f = function (n1, n2) {                    // 可以搞多态
    return n1 + n2
}

// 数组（有类型的数组）
let arr: stirng[];                         // 必须是字符串数组（坏耶）
let arr: Array<number>;                    // 必须是数字数组

// 元组
let t: [string, string]                    // 定长为 2 的字符串数组
let t: [number, string]                    // 可以是不同类型

// 枚举（可以手动指定 index）
Enum Gender {
    Male,

if (g === Gender.Female)                   // 全等判断值

type myType = 1 | 2 | 3 | 4
let s: myType

2.1 Class 基础

// 定义 Person 类
class Person {
    // 实例属性：只能通过实例进行访问
    age:  number;
    // 只读属性：不允许修改
    readonly name: string;
    // 类属性：添加 static 的静态属性可以直接通过 className 访问
    static animal: string = "monkey";
    // 需要套娃的话要按照 readonly static 的顺序

    // 具有固定名称的构造函数，默认包含指向当前实例的 this 指针
    constructor(name: string, age: number) {
        this.name = name;
        this.age = age;
    }

    // 实例方法
    sayHello() {
        console.log("Hello, my name is", this.name);
    }
    // 类方法
    static tellType() {
        console.log("A person has a name & age.")
    }
}

// 创建实例
p = new Person('Seabee', 23);
// 调用实例方法
p.sayHello();
// 调用类方法
Person.tellType();

2.2 继承

// 使用关键词 extends，继承所有的 方法 & 属性
class Student extends Person {
    // 新增属性：学号
    studentId: string;
    // 构造函数：需要在第一句调用父类构造函数
    constructor(name: string, age: number, id: string) {
        super(name, age);
        this.studentId = id;
    }
    // 同名函数重写
    sayHello() {
        // 也可以用 super 调用父类同名方法
        super.sayHello();
        console.log("我是学生，送我！");
    }
}

2.3 抽象类与接口

抽象类
// 抽象类无法被实例化
abstract class Animal {
    name: string;
    // 可以有已实现的方法
    constructor(name) {
        this.name = name
    }
    // 也可以有抽象方法（只能定义在抽象类中，子类必须重写以实现）
    abstract sayHello(): void;
}
接口
// 用于定义类的结构：至少需要包含的属性和方法
// 也可以作为类型被声明（但是类型别名只能声明一次）
interface Ducker {
    name: stirng;
    age: number;
}
// 可以多次定义同名借口，实际内容取并集
// 属性和方法都不能被赋值
interface Ducker {
    gender: string;
    swim(): void;
}

const duck: Ducker {
    name; 'Duke',
    age: 2,
    gender: 'female'
}

// 类实现接口使用关键字 implements
class Swan implements Ducker {
    // 不同于继承：需要手抄一遍全部成员
    name: string;
    age: number;
    gender: string;

    swim() {
        console.log("A swan is swimming...")
    }
}

2.4 属性封装

class sample {
    // 属性默认是 public，在任意位置可读可写
    public name: string;

    // 私有属性只能在类内部（通过 getter / setter）读写
    // 即便是子类也无法通过 this 指针访问（实例中有分配对应的空间）
    private _age: number;
    // 使用 get + 属性名 可以为对应属性设置 get 方法 => p.name == p.name()
    get age() {
        return this._age;
    }
    // 使用 p.age = 5 == p.age(5)
    set age() {
        if (age < 0) return;
        this._age = age;
    }

    // 子类中可以访问修改
    protected id: string;

    // 也可以直接在构造函数中定义属性及其可见度
    constructor(public name:string, private age: number) {
        this.name = name;
        this.age = age;
    }
}

2.5 范型

// 对于不确定类型的 函数/类，可以使用范型
// 在尖括号中定义范型关键字，具体类型在函数执行是确定
function calc<T>(a: T): T {
    return a;
}

// 使用
calc(10);             // 自动推导为 T = number
calc<string>('hello') // 手动指定   T = string

// 我们甚至可以一次指定多个范型（坏耶）
function fn<T, K>(a: T, b: K): T {
    return a;
}
fn<number, string>(123, '123');

// 限制范型类别范围（指定实现某接口）
interface Int {
    length: number
}
function size<T extends Int>(a: T): number{
    return a.length; // 由于 T 类型实现了 Int 接口，因此一定具有属性 length
}

// 用在类定义上更加抽象
class tmp<T> {
    id: T;
    constuctor(id: T) {
        this.id = id;
    }
}
const t = new tmp<string>('helo')
