    1873 计算特殊奖学金

    表: Employees
+-------------+---------+
| 列名        | 类型     |
+-------------+---------+
| employee_id | int     |
| name        | varchar |
| salary      | int     |
+-------------+---------+
employee_id 是这个表的主键。
    • 写出一个SQL 查询语句，计算每个雇员的奖金。如果一个雇员的id是奇数并且他的名字不是以'M'开头，那么他的奖金是他工资的100%，否则奖金为0。
    • 返回的结果集请按照employee_id排序。

    Solution 

    SELECT employee_id,
      (CASE 
       WHEN mod(employee_id,2)!=0 AND name NOT LIKE "M%" THEN salary
       ELSE 0
       END) bonus
FROM Employees
ORDER BY employee_id;

    字符串通配符

    • _ 用于替代单个字符，%用于替代0或多个字符
    • 需要配合 key_name LIKE/NOT LIKE 'string'语法进行使用

    CASE_WHEN语法

    (CASE
    WHEN condition_1 THEN return_val_1
    WHEN condition_2 THEN return_val_2
    ELSE return_val_n
END) AS key_name
    - 判断条件里可以嵌套 OR / AND - 返回值可以是 数字/字符串

    MOD_求余

    • 语法是 MOD( key, 除数)
    • 可以通过 == / != 判断计算结果是否符合要求

    627 变更性别

    Salary 表：
+-------------+----------+
| Column Name | Type     |
+-------------+----------+
| id          | int      |
| name        | varchar  |
| sex         | ENUM     |
| salary      | int      |
+-------------+----------+
id 是这个表的主键。
sex 这一列的值是 ENUM 类型，只能从 ('m', 'f') 中取。
    • 请你编写一个 SQL 查询来交换所有的 'f' 和 'm' （即，将所有 'f' 变为 'm' ，反之亦然），仅使用 单个 update 语句 ，且不产生中间临时表。
    • 注意，你必须仅使用一条 update 语句，且 不能 使用 select 语句。

    Solution 1 _ ASCII \(\(\because x \in \set{A,B} \ \therefore \ \neg{x}= (A + B) - x\)\) 

    UPDATE Salary SET sex = CHAR(ASCII('f') + ASCII('m') - ASCII(sex));
这边用 char() & ascii() 转换了一下 编码 & 字符

    Solution 2 _ CASE 

    UPDATE salary
SET sex = (CASE sex 不知道为啥这里要加 sex，总之就是分类讨论
           WHEN 'm' THEN 'f' 
           ELSE 'm'
           END);

    Solution 3 _ IF * 拿来当类似于 ?: 的三元运算符使了，大概就是 IF(condition, ture_rtn, false_rtn) 

    UPDATE salary SET sex = if(sex='m', 'f', 'm');

    196 删除重复的电子邮箱

    表: Person
+-------------+---------+
| Column Name | Type    |
+-------------+---------+
| id          | int     |
| email       | varchar |
+-------------+---------+
id是该表的主键列。
    • 编写一个 SQL 删除语句来 删除 所有重复的电子邮件，只保留一个id最小的唯一电子邮件。
    • 以 任意顺序 返回结果表。 （注意： 仅需要写删除语句，将自动对剩余结果进行查询）

    主要的坑在于： - 不能先从自己身上 SELECT 再以此为依据删自己的数据 - 需要为中间表赋别名 - 赋了别名的 SELECT 结果被视为 TABLE 而非 SET - 要使用 NOT IN 比较的话需要重新 SELECT

    Solution 

    DELETE FROM Person
WHERE id NOT IN
    (SELECT ID FROM
        (SELECT MIN(id) AS id
         FROM Person
         Group By email) Person2);

    1667 修复表中的名字

    表： Users
+----------------+---------+
| Column Name    | Type    |
+----------------+---------+
| user_id        | int     |
| name           | varchar |
+----------------+---------+
user_id 是该表的主键。
该表包含用户的 ID 和名字。名字仅由小写和大写字符组成。
    • 编写一个 SQL 查询来修复名字，使得只有第一个字符是大写的，其余都是小写的。
    • 返回按 user_id 排序的结果表。

    Solution 

    SELECT
    user_id,
    CONCAT(
        UPPER(LEFT(name, 1)),
        LOWER(RIGHT(name, LENGTH(name) - 1))
    ) AS name
FROM Users
ORDER BY user_id;

    字符串处理函数

    • 字符串连接函数 CONCAT(str1, str2)
    • 大小写转换函数 UPPER(str) / LOWER(str)
    • 取左右子串函数 LEFT(origin_str, len) / RIGHT(origin_str, len)
    • 取子串函数 SUBSTR(str, ini_pos, end_pos) 缺省 end_pos 参数时，默认取 ini - end
    • 字符串长度计算 LENGTH(str)

    1484 按日期分组销售产品

    表 Activities：
+-------------+---------+
| 列名         | 类型    |
+-------------+---------+
| sell_date   | date    |
| product     | varchar |
+-------------+---------+
此表没有主键，它可能包含重复项。（也可能包含 date & product 均相同的记录）
此表的每一行都包含产品名称和在市场上销售的日期。
    • 编写一个 SQL 查询来查找每个日期、销售的不同产品的数量及其名称。
    • 每个日期的销售产品名称应按词典序排列。
    • 返回按 sell_date 排序的结果表。

    Solution 

    SELECT
    DISTINCT sell_date,
    COUNT(DISTINCT product) AS num_sold, 
    // 因为可能有重复记录，所以不能直接对date 进行计数
    GROUP_CONCAT(
        DISTINCT product
        ORDER BY product
        SEPARATOR ',') AS products
FROM Activities
GROUP BY sell_date
ORDER BY sell_date;

    DISTINCT

    • DISTINCT 用于筛选 非重复 字段值集合（直接 SELECT 会产生重复项）
    • UNIQUE 用于唯一性约束（不能在 SELECT 里面修饰字段）

    GROUP_CONCAT

    • GROUP_CONCAT(exp) 函数用于将集合 exp 中的 not null 取值相连
      • 默认分隔符为 ,
      • 详细语法（方括号内均为可选值） GROUP_CONCAT([DISTINCT] key [ORDER BY ASC/DESC key][SEPARATOR] 'str')

    1527 患有某种疾病的患者

    患者信息表： Patients
+--------------+---------+
| Column Name  | Type    |
+--------------+---------+
| patient_id   | int     |
| patient_name | varchar |
| conditions   | varchar |
+--------------+---------+
patient_id （患者 ID）是该表的主键。
'conditions' （疾病）包含 0 个或以上的疾病代码，以空格分隔。
    • 写一条 SQL 语句，查询患有 I 类糖尿病的患者 ID （patient_id）、患者姓名（patient_name）以及其患有的所有疾病代码（conditions）。
    • I 类糖尿病的代码总是包含前缀 DIAB1 。
    • 按 任意顺序 返回结果表。

    Solution

    妙就妙在，我们不用对 conditions 进行拆分，并一个个对比字串 这里用的还是 LIKE 语法，但是要注意 开头/非开头 两种情况

    SELECT patient_id, patient_name, conditions
FROM Patients
WHERE conditions is not null
    and
    (conditions like 'DIAB1%'    // 开头
        or 
    conditions like '% DIAB1%'); // 非开头

    1795 每个产品在不同商店的价格

    表：Products
+-------------+---------+
| Column Name | Type    |
+-------------+---------+
| product_id  | int     |
| store1      | int     |
| store2      | int     |
| store3      | int     |
+-------------+---------+
这张表的主键是product_id（产品Id）。
每行存储了这一产品在不同商店store1, store2, store3的价格。
如果这一产品在商店里没有出售，则值将为null。
    • 请你重构 Products 表，查询每个产品在不同商店的价格，使得输出的格式变为(product_id, store, price) 。如果这一产品在商店里没有出售，则不输出这一行。
    • 输出结果表中的 顺序不作要求 。

    Solution 

    SELECT *
FROM (
    SELECT product_id, 'store1' AS store, store1 AS price FROM Products
    UNION
    SELECT product_id, 'store2' AS store, store2 AS price FROM Products
    UNION
    SELECT product_id, 'store3' AS store, store3 AS price FROM Products
) AS tmp
WHERE price IS NOT NULL;

    UNION

    • UNION 会去除同样的记录
    • UNION ALL 会保留 duplicates

    行列转换

    • 行 to 列：CASE...WHEN/IF
    • 列 to 行：UNION
