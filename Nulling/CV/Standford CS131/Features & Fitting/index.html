Features & Fitting

1 RANSAC

一种用于 line detection 的 model fitting 方法

Fitting - Search in Param Space

我们选用某种 parametric model 来代表 a set of features：

  • Membership criterion is NOT LOCAL

    我们没办法只看单个点就确定他是否符合给定的模型

  • 三个主要问题：

    1. What model represents this set of features BEST?
    2. Which of several model instances gets which feature?
    3. How many model instances are there?

以直线探测为例：

  • 每个像素属于哪一条直线（活着根本不属于任何一条）
  • 如果我们只探测到了一条直线的部分，怎么找到连接各部分的 bridge 呢
  • 噪声可能被识别为 edge point，并对直线的方向产生影响（如何排除呢）

Voting

It is NOT feasible to check ALL combinations of features by fitting a model to each possible subset.

我们可以让 feature 为所有与之相容的模型投票，从而找到得票最多的 parameters

  • 噪声 & clutter feature 也会产生投票，但这些票数一般和 good feature 不相干
  • 由于模型可以跨越多个 fragment，在未观测到部分特征时也行得通

RANdom SAmple Consensus

  • 为了排除 outliner 的影响，我们计划找出 inlier，并仅用这些数据进行拟合

  • 在尝试通过 outliner 计算 current fit 时，产生的结果将得到较低的支持度

  • Loop

    1. 随机选择一组 seed points 来建立最初的估计（对于直线就是抓俩点出来）
    2. 找到符合本次估计的 inliers（到给定直线距离小于阈值）
    3. 如果 inlier 的足够，重新基于本次选出的 inlier 计算 least-square estimate
    4. 重复 2-3 直至找到具有 最多 inlier 的估计

  • How many samples should we choose?

    • 假设 inlier 所占的百分比应为 \(w\)，我们需要 \(n\) 个点确定一个假设（两点确定直线），并检定了 \(k\) 个样本
    • 单个样本正确的概率为 \(w^n\)
    • 所有 \(k\) 个样本均检定失败的概率为 \((1-w^n)^k\)

    因此，选择 top \(k\) 个样本即可使得检定失败的概率变得非常小

  • 存在的问题

    如果不希望计算成本爆炸 RASAC 只能处理 outlier 占比较小的情况（但很多情况下 outlier 占比都比较大）

    我们可以使用 vote 策略（Hough Transform）来处理 outlier 占比较高的情况

2 Local Invariant Features

2.1 Motivation

进行图片匹配时，我们往往抓住 主要矛盾 —— 一些局部特征。

事实上，match only local regions 在诸如 Occlusions（遮挡剔ß除）, Articulations（关节识别）, Intra-category Variations（种内差异比较）等领域上具备比 global representation 更强的鲁棒性。

General Approach

  1. Find a set of distinctive keypoints
  2. Define a region around each keypoint
  3. Extract & Normalize region content
  4. Compute a local descriptor from normalized region
  5. Match local descriptors

2.2 Requirements & Invariances

  • 可能存在的问题

    1. 我们需要在两幅图片中找到对应的关键点（不能尝试通过驴头匹配驴嘴）
    2. 两幅图中的关键点应该一一对应（不是田忌赛马）
    3. 可能面临一些几何变化（比如旋转和拉伸）
    4. 可能面临一些色相变化（比如拍摄光线导致的影响，一般用 linear transformation 解决）

  • 对应的要求

    • Region extraction needs to be repeatable & accurate

      • Invariant to translation, rotation, scaling ...
      • Robust / Covariant to OUT-OFF-PLANE(affine) transformations
      • Robust to lighting variations, noises, blur, quantization

    • Locality: Features are local -> robust to occlusion & clutter

    • Quantity: A sufficient number of regions is required to cover the object
    • Distinctiveness: The region should contain interesting structure
    • Efficiency: Close to real-time performance

3 Keypoint Localization

  • 实现关键点定位的主要目标有：

    • Repeatable detection
    • Precise localization
    • Interesting content

    => 我们可以尝试寻找 2-dimensional signal changes

  • Corner 探测的主要特点：

    在区域的角落部分，图片的梯度有着 2 or more dominant directions

    Corners are repeatable & distinctive => 这正是关键点定位的目标 !

  • Corner's criteria

    • Locality: 我们应当可以通过一个 small window 对角落进行探测

    • Good Localization: 向 任意方向 移动 window 都应该产生巨大的 intensity 变化

    我们可以考虑将 window 框选的区域划分为以下三种：

    1. Flat：向 任意方向 移动 window 都不会产生变化
    2. Edge：在 沿 edge 方向移动不会产生变化
    3. Corner：向 所有方向 移动均会产生 显著变化

Harris Corner Detector

  • 对于单个像素点的 intensity difference，我们可以用 \(I(x+u, y+v) - I(x,y)\) 计算。

  • Harris Detector 解决了对整个 window 变化量进行计算的问题。

对于偏移量 \([u,v]\)，整个 window 产生的 shift 为：

\[ E(u,v) = \sum_{x,y}w(x,y)[I(x+u, y+v)-I(x,y)]^2 \]

其中：

  • \(w(x,y)\) 被称为 window function，可以是 阶梯函数 / Gaussian 函数
  • \(I(x+u,y+v)\) 被称为 shifted intensity
  • \(I(x,y)\) 是 window 原本位置的 intensity

这一变化量也可以用 Taylor Expansion 进行估计：

\[ E(u,v) \approx [u,v] M \left[ \begin{array}{1} u\\ v \end{array} \right] \]

其中 \(M\) 是一个 2 * 2 的矩阵，可以通过 image derivatives 计算得出：

\[ M = \sum_{x,y}w(x,y) \left[ \begin{array}{2} I^2_x & I_xI_y\\ I_xI_y & I_y^2 \end{array} \right] = \sum \left[ \begin{array}{1} I_x\\ I_y \end{array} \right] [I_x, I_y] \]
Axis-Aligned Corner
\[ M = \left[ \begin{array}{2} \lambda_1 & 0\\ 0 & \lambda_2 \end{array} \right] \]

这代表：

  • Dominant gradient direction align with \(x\ or\ y\) axis（直角边框）
  • 如果 \(\lambda_1 \& \lambda_2\) 都接近于 0 那么这不是 corner => 我们需要较大的 \(\lambda\) !
General Case

首先 \(M\) 是对称的，我们可以基于 Eigenvalue Decomposition(特征值分解) 将其变换为：

\[ M = R^{-1} \left[ \begin{array}{2} \lambda_1 & 0\\ 0 & \lambda_2 \end{array} \right] R \]

我们可以把矩阵 \(M\) 想象成一个长短轴由特征值决定、方向由 \(R\) 确定的椭圆：

  • 我们可以使用 \(M\) 的特征值来对 image 中的点进行分类：

  • Corner Response Function

    \[ \theta = det(M) - \alpha trace(M)^2 = \lambda_1\lambda_2 - \alpha(\lambda_1 + \lambda_2)^2 \]
    • 一般避免计算详细的特征值，\(\alpha\) 是介于 \([0.04,0.06]\) 区间内的常数
    • edge 对应的 \(\theta \lt 0\)，而 corner 对应的 \(\theta \gt 0\)
Window Function \(w(x,y)\)

  1. Uniform Window

    Sum over square window - window 内取 1，外取 0

    \[ M = \sum_{x,y} \left[ \begin{array}{2} I^2_x & I_xI_y\\ I_xI_y & I_y^2 \end{array} \right] \]

    NOT rotation invariant

  2. Smooth with Gaussian

    使用高斯平滑计算 window 内外的 weighted sum，可以应对 rotation（旋转不变）

    \[ M = g(\sigma) * \left[ \begin{array}{2} I^2_x & I_xI_y\\ I_xI_y & I_y^2 \end{array} \right] \]
Algorithm
  1. 计算 image derivatives：\(I_x,I_y\)
  2. 计算 square of derivatives：\(I_x^2,I_y^2\)
  3. 使用 Gaussian Filter：\(g(I_x^2),g(I_y^2),g(I_xI_y)\)

  4. 使用 Corner Function：

    $$
\theta = det[M(\sigma_I,\sigma_D)] - \alpha[trace(M(\sigma_I,\sigma_D))]^2 = g(I_x^2)g(I_y^2)-\alpha[g(I_x^2) + g(I_y^2)]^2
$$

  5. 采用 Non-maximum Suppression

其中 \(M(\sigma_I,\sigma_D)\) 为 autocorrelation matrix:

\[ M(\sigma_I,\sigma_D) = g(\sigma_I) * \left[ \begin{array}{2} I_x^2(\sigma_D) & I_xI_y(\sigma_D) \\ I_xI_y(\sigma_D) & I_y^2(\sigma_D) \end{array} \right] \]

其中：

  • \(\sigma_D\) 是用于计算 derivate 用的高斯变换
  • \(\sigma_I\) 是用于 window function 的高斯变换
Summary
  • 具有 旋转不变性
  • 不具备放缩不变性 —— 放大会导致原本的 corner 被判定为 edge
