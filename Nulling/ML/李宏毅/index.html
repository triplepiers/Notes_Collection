李宏毅机器学习

NLP

Self-Attention

解决 Sequence Labeling 问题：给每一个 vec 贴标签

传统模型解决的问题如下：

\[ 1\ Vec \rightarrow Model \rightarrow \text{数值 / 类别} \]

What If

  • 输入：变长 · 向量 · 序列 ？
    • 一条长度不定的句子
    • 一条长度不定的语音
  • 输出：
    • 给每一个 vec 贴上各自的 label (Seq Labeling)
    • 对整个 sequence 给出一个 label
    • 自己决定具体给多少的 label (Seq2Seq)

传统的 Fully-Connected Network 并不会考虑上下文信息，每个输入对应了固定的输出

对于序列 I saw a saw，两个 saw 会被给出同样的标签 => 但显然它们具有不同的词性

  1. Window: Consider the neighbor

    同时输入 t-1, t, t+1 来判断 t 的类别（具体考虑多少个上下文由窗口大小决定）

    Window 不能顾及整个 seq 的信息：我们无法确定最大的 seq 长度 & 会显著增加参数

  2. Self-Attention: Consider WHOLE Sequence

    可以通过交替使用 self-attention 和 fully-connected network 分别处理 全局 / 局部 信息

    Is Relevant

    对于特定位置的输入向量 \(x_i\)，找出整个序列 \(\{X\}\) 中与之相关的 所有向量

    • 使用 \(\alpha\) 来衡量两个向量 \((x_i, x_j)\) 间的相关程度

      可以采用 dot-product / additive 等方案计算

    • 考虑位置 \(x_i\)：

      1. 计算 query \(q_i = W_q \cdot x_i\)
      2. 计算其他所有位置的 key \(k_j = W_k \cdot x_j\)
      3. 计算 \((x_i,x_j)\) 之间的 attention score \(\alpha_{i,j} = q_i \cdot k_j\)
      4. 对 \(\alpha\) 做 SoftMax（或者其他激活函数）得到 \(\alpha'\)
      5. 计算所有位置的 value \(v_j = W_j \cdot x_j\)
      6. 计算 \(x_i\) 的最终结果 \(b_i = \sum \alpha_{i,j}' \cdot v_j\)

      上述过程使用矩阵乘法描述如下：

      \[ \begin{align*} Q &= W_q \cdot Input \\ K &= W_k \cdot Input \\ V &= W_v \cdot Input \\ A &= K^T \cdot Q \\ A' &= activation(A) \\ Output &= V \cdot A' \end{align*} \]

      需要学习的参数只有 \(W_q, W_k, W_v\)

  • CNN 其实可以看作简化版的 Self-Attention => 只考虑了感受野内的输入相关性

  • 也可以认为 Self-Attention 是一个自动学习感受野范围的 CNN

Multi-head Self-Attention

事物之间的联系是多种多样的 => 我们需要多个 \(q\) 来判断两者是否相关

我们可以通过超参数 \(n\) 决定具体计算多少种联系（于是参数多了 \(n\) 倍）

每种联系独立计算，可以得到 \(b_{i,1}, ... b_{i,n}\) => 对其进行 concat：

\[ b_i = W_O \cdot \left[ \begin{array}{1} b_{i,1} \\ b_{i,2} \\ ... \\ b_{i,n} \end{array} \right] \]

Positional Encoding

Self-Attention 并没有关注 Position info.

你可以自己把位置信息塞进去

  • 为每个位置 \(i\) 设置 unique 的 Positional Vec \(e_i\)

    可以 手动指定(hand-crafted) / learn from data

  • 对 \(a_i' = a_i + e_i\) 进行 Self-Attention 计算

Truncated Self-Attention

有点类似于批处理

在处理语音信号时，因为采样频率很高 => 输入会变的长长长 ...

=> Attention in a range：只考虑对 指定 window size 内的输入应用 Self-Attention

解决 Sequence to Sequence 问题：自己决定输出序列长度

$$ [x_1, x_2, ... x_n] \rightarrow Encoder \rightarrow [h_1, h_2,...h_n] $$

输入输出等长

  • Transformer 使用 Self-Attention 进行实现（其实 RNN 和 CNN 也行）

  • Transformer Block 采用了两处残差：

Masked Self-Attention

计算 \(b_i\) 时，仅考虑 \(x[0:i]\) => 输出是一个个蹦出来的

When to STOP?

如何让机器自行决定 Output Sequence 的长度 => 将 Output Vec 的长度改为 len(dict) + 1

我们添加对 END Token 的概率推测，在 SoftMax 选出 END Token 时终止

所以 AT Decoder 的实际输入输出序列如下：

\[ \begin{align*} Input &= [BEGIN, x_1, x_2, ...., x_n] \\ Output &= [y_1, y_2, ... ,y_m, END] \end{align*} \]
Non-Autoregressive (NAT)

不同于 AT 一个一个蹦输出，NAT 一次性产出整个 Output Sequence

NAT Decoder 的实际输入输出序列如下：

\[ \begin{align*} Input &= [BEGIN, BEGIN, ..., BEGIN] \\ Output &= [y_1, y_2, ... ,y_m] \end{align*} \]

When to STOP?

  1. 额外训练一个用于预测输出长度的 predictor
  2. 直接蹦一个很长的序列，然后在 END 处截断（产出 300 个，但只打印前 28 个）

NAT Decoder 的优点如下：

  • 支持并行计算（蹦一整个，AT 需要返回 \(y_{t-1}\)）
  • 输出长度可控

但 NAT 的效果一般没有 AT 那么好

Cross-Attention

用于联系 Encoder & Decoder

对于 Masked Self-Attention 产出的每一个 \(y_i^2\)：

  • 取对应的 \(y_i^2\) 与 Encoder Output 各位的 \(k_j^1\) 计算 attention score \(\alpha_j^1\)
  • 使用 attention score 与 Encoder Output 各位的 \(v_j^1\) 计算最终得分 \(v_i^2\)
  • 使用 \(v_i^2\) 作为 Decoder Fully-Connected Network 的输入

Training

  1. 优化目标

    • Loss = 最小化两者的 cross entropy
      • Transformer 输出的每一个 vec 其实都是一个 概率向量
      • 每一位的 “标准输出” 也可以用 one-hot 变成一个等长向量

    • 调参时使用了 Teacher Forcing：使用 Ground Truth 作为 Decoder 的输入

      这会导致 Exposure Bias —— 预测的上一个输出不一定是对的，且会返回作为输入

      Scheduled Sampling：考虑在 training 的过程中掺杂部分 wrong input

  2. Copy Mechanism：输出其实可以照搬部分输入（比如用户名）

  3. Guided Attention

    指定 attention 曲线的变化规律（文本转语音的注意力必然是自左向右的）

  4. Beam Search

    Greedy Decoding 不能保证全局最优 & 穷举不太实际

    Not always work，有时加一些 random 可能会更好
