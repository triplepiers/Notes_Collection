AutoGrad

自动梯度计算

Sample：求函数 \(y = 2x^Tx\) 关于列向量 \(x\) 的梯度

from mxnet import autograd, nd

# 初始化变量 x
x = nd.arrange(4).reshape((4,1))

# 申请用于存储梯度的内存
x.attach_grad()

# 调用 record 函数记录与梯度相关的计算（默认不记录以减少开销）
with autograd.record():
    y = 2* nd.dot(x.T, x)

# 因为 y = (4,1) * (1,4) = (1,1) 为标量，可以直接用 backward() 求梯度
# 若 y 并非标量，则会先调用 y.sum()
y.backward()

# 验证 y ?= 4x
assert (x.grad - 4*x).norm().asscalar() == 0

# 输出梯度计算结果
print(x.grad)

即便函数计算中包含了 while 和 if 控制语句，我们也可能对变量求梯度：

  • 函数 f(a) 的输出取决于 a 的值

    def f(a):
    b = a * 2
    while b.norm().asscalar() < 1000:
        b = b * 2
    if b.sum().asscalar() > 0:
        c = b
    else:
        c = b * 100
    return c

  • 我们同样可以使用 backward() 求梯度

    a = nd.random.normal(shape=1)
a.attach_grad()
with autograd.record():
    c = f(a)
c.backward()

# 由于 f(a) = a * x -> f(a)' = x = c/a
assert a.grad == c / a

Linera Regression 线性回归

使用一条直线来拟合训练数据，并使用参数 \((w,b)\) 进行预测：

\[ \hat{y} = wX + b \]

优化目标：找到一组 \((w,b)\) 以最小化平方误差项：

\[ \sum(\hat{y}_i - y_i)^2 \]

我们可以将 线性回归 模型视为一个使用 \(g(x) = x\) 作为激活函数的单层神经网络

1 生成数据集

我们根据以下公式生成训练用的数据集：

\[ y[i] = 2 * x[i][0] - 3.4*x[i][1] + 4.2 + \text{noise} \]
  • 噪声符合 \(\mu=0,\sigma=0.1\) 的正态分布
  • 理想情况下参数 \((w,b) = ([2,-3.4]^T, 4.2)\)
from mxnet import ndarray as nd

n_feature  = 2    # 每个样本含有 2 个 feature
n_example = 1000 # 生成 m=1000 的数据集

# 理想情况下的参数
true_w = [2, -3.4]
true_b = 4.2

# 生成 2x1000 个 features
x = nd.random_nromal(shape=(n_examples, n_feature))

# 按公式生成 label
y = true_w[0] * x[:, 0] + true_w[1] * x[:, 1] + true_b
y += 0.01 * nd.random_normal(shape=y.shape) # 添加正态分布的噪声

2 小批量读取数据

封装以下函数，每次返回 batch_size 个 随机样本

# features 对应训练集中的 X
# labels   对应训练集中的 Y
def data_iter(batch_size, features, labels):
    n_example = len(features)
    indices = list(range(n_example))
    # 读取顺序是随机的
    random.shuffle(indicies)
    for i in range(0, n_example, batch_size):
        j = nd.array(indices[i: min(i+btach_size, n_example)])
        yield features.take(j), labels.take(j) # 抽取下标 j 对应的样本

3 定义并训练模型

from mxnet import autograd

# 初始化参数 w,b
w = nd.random.normal(scale=0.01, shape=(n_feature, 1))
b = nd.zeros(shape=(1,))

# 由于之后需要对 w,b 进行梯度下降，此处需要创建对应的梯度
w.attach_grad()
b.attach_grad()

# 定义线性回归模型（预测逻辑）
def linreg(X, w, b):
    return nd.dot(X, w) + b

# 定义损失函数（均方误差）
def squared_loss(y_hat, y):
    # 统一两者的 shape 避免自动广播
    return (y_hat - y.reshape(y_hat.shape)) ** 2

# 随机梯度下降（参数更新）
def SGD(params, alpha, batch_size):
    for p in params:
        p[:] = p - alpha * p.grad / batch_size

# 训练模型
epochs = 5
alpha = 0.001
for e in range(epochs):
    total_loss = 0
    for data, label in data_iter(batch_size, x, y): # 每次对一个 batch 进行梯度下降
        # 前向传导计算误差
        with autograd.record():
            output = linreg(data) # 使用当前参数进行预测
            loss = square_loss(output, label)
        # 反向传播进行梯度下降
        loss.backward()
        SGD(params, alpha, batch_size)
        # 计算总误差
        total_los += nd.sum(loss).asscalar()
    print("in Epoch[%d], average loss = %f" % (e, total_loss/n_example))

L2 范数正则化

对 绝对值较大 的参数进行惩罚，需要额外定义正则化系数 \(\lambda\)

L2 范数正则化项定义如下（此时 Cost = Loss + L2）：

\[ L_2 = \lambda \cdot \sum_{p \in params} \|p\|^2 \]

在 测试模型时 需要令 \(\lambda = 0\)

正则化的 Gulon 实现
model = nn.Sequential()
model.add(nn.Dense(1))
model.initialize(init.Normal(sigma=1))

# 对 trainer 添加超参数 wd(weight decade) 即可
# 对参数 W 进行正则化
trainer_w = gluon.Trainer(
    model.collect_params('.*weight'), 'sgd',
    {'learning_rate': 0.01, 'wd': wd}
)
# 不对参数 b 进行正则化
trainer_b = gluon.Trainer(
    model.collect_params('.*bias'), 'sgd',
    {'learning_rate': 0.01}
)

# 此时，每一轮训练应该对 w,b 分别调用 step() 以同时更新

CNN 卷积神经网络

Flatten 操作 => 卷积救一下啊！！！

  1. 会破坏像素中包含的语义
  2. 全连接导致输入过大

  1. 卷积层

    • 卷积核 kernel / filter：一个比较小的滑动窗口
    • 偏移量 bias：类似于 \(wX + b\) 的操作

    对于多通道输入，一次卷积后的结果为： $$ conv(data, w, b)[:,i,:,:] = conv(data, w[i,:,:,:]m,b[i]) $$

    其中 \(w[i]\) 为各通道的权重（卷积核），\(b[i]\) 为该通道对应的偏移量

  2. 池化层 Pooling

    进一步减少 feature 的数量，降低微小变化对模型的干扰

    • 最大池化：选用一个窗口中的 max 值
    • 平均池化：选用一个窗口中的 avg 值

  3. 全连接层

    会对输入做 Flatten 操作

    Sample

  4. 数据集：还是 FashionMNIST

  5. 模型：LeNet = 2 * 卷积层 + 2 * 全连接层

Dropout 丢弃法解决过拟合

用于多层神经网络

DropOut 就是对 输入层/隐藏层 做以下操作：

  1. 随机选择该层的 部分输出
  2. 将这些元素置 0
  3. 对 非丢弃元素 进行 Flatten 操作
DropOout 实现
# drop_probability 定义了单个元素被丢弃的概率

def dropout(x, drop_probability):
    keep_probability = 1 - drop_probability
    assert 0 <= keep_probability <= 1
    # a) 全部丢弃
    if keep_probability == 0:
        return x.zeros_like()
    # b) 随机丢弃部分元素
    mask = nd.random.uniform(0, 1.0, x.shape) < keep_probability
    scale = 1 / keep_probability # 令 E(dropout(x)) == x
    return mask * x * scale
  • DropOut 的本质是：每次只激活全连接层中的部分节点
  • 我们可以为每一层单独设置 drop_probability，一般越靠前的丢弃概率越低

BatchNorm 批量归一化

虽然最终参数不一定更好，但是收敛更快

  • 我们通常会将 原始输入 的每一个 feature 都进行归一化 => 变成 \(avg = 0, \sigma = 1\) 的格式

  • 但在深度模型中，乘法效应 可能使得不同层的输入数据范围相差极大=> 采用相同的学习率会导致不同层的梯度下降效率极不一致

    于是我们：对 每层的输入 都进行归一化

考虑一个小批次 \(B = (x_1, ... , x_m)\)：

  • 需要学习：拉升参数 \(\gamma\)，偏移参数 \(\beta\)
  • 归一化结果为： \(y_i = BN_{\gamma,\beta}(x_i)\)

此外，定义：

\[ \begin{align*} \mu_B \leftarrow \frac{1}{m} \sum x_i \tag{均值}\\ \sigma_B^2 \leftarrow \frac{1}{m}\sum(x_i - \mu_B)^2 \tag{方差}\\ \hat{x_i} \leftarrow \frac{x_i - \mu_B}{\sqrt{\sigma_B^2 + \varepsilon}} \tag{归一化的 X}\\ y_i \leftarrow y \hat{x_i}+\beta = BN_{\gamma,\beta}(x_i) \end{align*} \]

此外，在实现时需要区分全连接层和卷积层：

  • 全连接层：对 每一个 batch 进行归一化
  • 而为卷积层：对每一个 channel 进行归一化，并保持四维形状
def batch_norm(x, gamma, beta, eps=1e-5):
    assert len(x.shape) in (2,4)

    if len(x.shape) == 2: # 全连接层: 对每个 feature 求均值
        mean = x.mean(axis=0)
        variance = ((x - mean)**2).mean(axis=0)
    else:                 # 二维卷积: 对每个 channel 求均值
        mean = x.mean(axis=(0,2,3), keepdims=True)
        variance = ((x - mean)**2).mean(axis=(0,2,3), keepdims=True)

    # 均一化
    x_hat = (x - means) / nd.sqrt(variance + eps)

    # 拉升与偏移
    return gamma.reshape(mean.shape) * x_hat + beta.reshape(mean.shape)

Network In Network

当我们把 Dense 层和 Conv 层进行混编时，会出现一个难题：

  • Dense 层需要 2D 的输入输出，而 Conv 则要求 4D
  • 直接 \(4D \rightarrow 2D\) 产生的全连接层有过多的参数

  • 像素之间共享权重

Inception

Inception 是 GoogLeNet 的主要组成构件，由四个并行 Conv 块构成。对于每一个 input，都有 4 条 path 通向输出端（Concat）：

graph TB
    A[input]-->B[1x1 Conv]
    A-->C[1x1 Conv]-->D[3x3 Conv]
    A-->E[1x1 Conv]-->F[5x5 Conv]
    A-->G[3x3 MaxPool]-->H[1x1 Conv]
    B-->I[Concat]
    D-->I
    F-->I
    H-->I
  • Path 1 ～ Path 3 分别使用 1x1，3x3，5x5 的 kernel 来提取不同 size 的特征
  • Path 2 ～ Path 4 中的 1x1 Conv 用于降低通道数量（避免 concat 产生的数据过大）
实现 Inception Block
class Inception(nn.Block):
    def __init__(self, n1_1, n2_1, n2_3, n3_1, n3_5, n4_1, **kwargs):
        super(Inception, self).__init__(**kwargs)
        with self.name_scope():
            # path_1
            self.p1_conv_1 = nn.Conv2D(
                n1_1, kernel_size=1, activation='relu'
            )
            # path_2
            self.p2_conv_1 = nn.Conv2D(
                n2_1, kernel_size=1, activation='relu'
            )
            self.p2_conv_3 = nn.Conv2D(
                n2_3, kernel_size=3, padding=1, activation='relu'
            )
            # path_3
            self.p3_conv_1 = nn.Conv2D(
                n3_1, kernel_size=1, activation='relu'
            )
            self.p2_conv_5 = nn.Conv2D(
                n3_5, kernel_size=5, padding=2, activation='relu'
            )
            # path_4
            self.p4_pool_3 = nn.MaxPool2D(
                pool_size=3, padding=1, strides=1
            )
            self.p4_conv_1 = nn.Conv2D(
                n4_1, kernel_size=1, activation='relu'
            )

    def forward(self, x):
        p1 = self.p1_conv_1(x)
        p2 = self.p2_conv_3(self.p2_conv_1(x))
        p3 = self.p3_conv_5(self.p3_conv_1(x))
        pr = self.p4_conv_1(self.p4_pool_3(x))

        return nd.concat(p1, p2, p3, p4, dim=1)

ResNet 残差网络

ResNet 有效解决了 深度卷积网络难以训练 的问题：在反向传播中，梯度通常会越来越小 => 这导致权重变化量减小，并导致深层网络训练缓慢。

在此之前，人们通常通过两种方案解决这个问题：

  1. 按层训练：先训练靠近 input 的层，缓慢增加更深的层（效果不太好）
  2. 使用更宽的层（而不是更深）=> 效果没有深层模型好

而 ResNet 通过增加 跨层连接 解决了反向传播梯度逐层减小的问题：

  • Layer 1 的输出不仅传递给了 Layer 2，并且与 Layer 2 的输出相加后进入 Layer 3

    => 这使得反向传播时，梯度可以 跳过 Layer 2 直接传递给 Layer 1

  • 之所以被称为 “残差网络” 是因为可以将这个结构看成 2 个网络只和（它们共享 Layer 1）：

    1. Layer 1：因为跳过了 Layer 2，所以更容仪训练
    2. Layer 1 -> Layer 2：上一部分没有拟合的 残差 由本部分拟合
graph LR
    A[Layer 1]-->B[Layer 2]-->C((+))-->D[Layer 3]-->E((+))-->F(result)
    A-->C
    C-->E
Residual Block

在 v2 中，作者将 Residual Block 的架构从 Conv -> BatchNore -> ReLU 改成了 BatchNorm -> ReLu -> Conv

  • ResNet 沿用了 VGG 中全部采用 3x3 Conv 的思想，但在卷积层和池化层中间加入了 BatchNorm 层以加速训练。

  • 每一次跨层连接将跨越两个卷积层。

  • 当输入输出的 channel 数量不一致时，使用 1x1 Conv 进行通道变化，并使用 strides=2 使长宽减半

我们可以根据上述定义实现 残差块：

class Residual(nn.Block):
    def __init__(self, channels, same_shape=True, **kwargs):
        super(Residual, self).__init__(**kwargs)
        # 输入输出是否大小一致
        self.same_shape = same_shape
        with self.name_scope():
            strides = 1 if same_shape else 2
            self.conv1 = nn.Conv2D(
                channels, kernel_size=3, padding=1, strides=strides
            )
            self.bn1 = nn.BatchNorm()
            self.conv2 = nn.Conv2D(
                channels, kernel_size=3, padding=1
            )
            self.bn2 = nn.BatchNorm()

            if not self.same_shape:
                self.conv3 = nn.Conv2D(
                    channels, kernel_size=1, strides=strides
                )

    def forward(self, x):
        out = nd.relu(self.bn1(self.conv1(x)))
        out = self.bn2(self.conv2(out))
        if not self.same_shape:
            x = self.conv3(x)
        return nd.relu(out + x)

DenseNet 稠密连接

其实就是把 ResNet 中的 add 操作改成了 concat 操作：

graph LR

    A-->C

  • 相比于 add，concat 操作 更完整的保留了底层输入 => 原封不动的拼接上去

  • 这同时也导致，每经过一个构件（Dense Block），输出的通道数都可能激增

通过交替串接 Dense Block 和 Transition Block，我们就可以得到一个不太爆炸的 DenseNet

  • 在每一张图片上选取多个 region，每个 region 都作为 CNN 的一个样本用于抽取特征。

  • 最后使用 分类器 对主体内容进行分类，并使用一个 回归器 得到准确的边框。

graph LR
    A(Image)-->B[Region 1]-->G[Resize]--same Size-->C[Conv]-->D[[SVMs 分类器]]
    C-->E[[BBox Reg 边缘盒回归]]
    A-->F[Region 2]-->G

其中：

  • 用于特征提取的 CNN 模型与 迁移学习 类似：将一个预训练模型去掉输出层，然后进行参数微调
  • 采用 多个 SVM 进行分类，每一个执行 IN/OUT 的而分类问题

但由于一张图片中可以选出非常多的 region，R-CNN 的预测效率较低

Fast R-CNN

主要通过两处改进来提升性能：

  1. R-CNN 中抽取的 region 可能存在大量 互相覆盖 的情况，对这些 region 重新抽取特征过于浪费

    => 先对原始图片抽取 feature，再选取 region

    => 只进行 一次特征提取

  2. 使用一个 SoftMax 进行多分类（而不是多个二分类）

graph LR

    A(Image)-->B[Conv]-->C[RoI Pooling]-->D[Dense]-->E[[Softmax]]
    D-->F[[BBox Reg]]
    B-->G[Search Regions]-->C

RoI Pooling 兴趣区域池化层

Region of Interest

  • 输入：feature & 一系列的 region
  • 过程：将每个 region 均匀划分为 \(n \times m\) 的小块，并对每一个小块做 MaxPooling
  • 输出：\(n \times m\) 向量（任何 shape 的输入都将产生固定大小的输出）
# 假设提取了 4x4 且 channel=1 的特征
x = nd.arrange(16).reshape((1,1,4,4))

# 使用五元组表示 region: (idx, x_min, x_max, y_min, y_max)
r1 = [1,0,0,2,2] # 大小为 3x3 的区域
r2 = [2,0,1,3,3] # 大小为 4x3 的区域

# RoI Pooling：输出 n_region x n_channel x n x m
input = nd.array(r1,r2)
nd.ROIPooling(x, input, pooled_size=(2,2), spatial_scale=1)

Faster R-CNN

R-CNN 和 Fast R-CNN 均采用 选择性搜索 来划分区域，这是一个性能瓶颈。

对此，Faster R-CNN 提出 Region Proposal Network (RPN) ：

  1. 对输入特征进行 padding=1, channel=256 的 3x3 卷积，使得每 9 个相邻像素都被映射为长 256 的 vec
  2. 以每个 pixel 为中心生成 k 个 长宽比 & 大小预设的 锚框

  3. 对每个 锚框，使用中心 pixel 对应的 256 维 vec 作为特征

    训练一个二分类器判断：是否包含感兴趣物体（或只有背景）

    训练一个回归器（输出四元组）：预测更加准确的边框

  4. 对于所有 \(nmk\) 个 锚框，选出其中 包含物体的项，将回归结果作为 Pooling 输入

graph LR
    A(Conv)-->B[nxm Feature]-->C[3x3 Conv]-->D[[SoftMax 二分类]]
    C-->E[[BBox Reg]]-->F[RoI Pooling]

Mask R-CNN

在 Faster R-CNN 的基础上加入了一个 pixel 级别的预测层 —— 全连接卷积网络（FCN）。

不仅对 单个锚框 预测包含物品种类和真实边框，还会对其包含的像素进行语义分割（输出像素级别的标注，而非方框）

graph LR
    A(Image)-->B[Conv]-->C[RoI Align]-->D[Dense]-->E[[Softmax]]
    D-->F[[BBox Reg]]
    B-->G[RPN]-->C
    C-->H[FCN]

SSD 单发多框检测器

在 R-CNN 中，Region Proposal 和 Classify 时两个独立的任务，SSD 将其统一成一个步骤以加速模型 —— 这也是被称为 “单发” 的原因

SSD 和 R-CNN 的主要区别如下：

  • 对于 锚框，不再先判断 是否包含物体，再对重定边框进行物体分类

    SSD 使用 n_class + 1 的多分类器完成 具体类别+背景 的多分类任务；同时不再使用 BBox Reg 进一步预测更加准确的边框，而是直接使用单个回归器预测 真实边框

  • 预测的对象不仅是 CNN 提取的 feature，还通过 Conv + Pooling 进一步缩小特征，从而达到多尺度预测

graph LR
    A(Image)-->B[Conv]-->C((Features))--"Multi-Scale"-->D[Conv]-->E((Smaller Ft))-->F[[Softmax]]
    E-->G[[BBox Reg]]

    C-->H[["Softmax (n+1)"]]
    C-->I[["BBox Reg (true bound)"]]

用于从原始图像中抽取特征的 Conv 可以选择前面用于图像分类的 CNN 模型，比如 ResNet

Anchor Box

  • 由于实际边框可能出现在图片中的任何位置，且拥有任意大小。为了简化计算，SSD 和 Faster R-CNN 均采用预设的 anchor box 作为搜索起点。

  • 对于输入的 每一个像素，以其作为中心采样若干个大小、比例不同的 锚框

假设输入的大小为 \(w \times h\)（不会同时应用 size & ratio）：

  • 给定大小 \(s \in (0,1]\)，生成的锚框形状为 \(ws \times hs\) （不超过原输入）
  • 给定比例 \(r \gt 0\)，生成的锚框形状为 \(w\sqrt{r} \times \frac{h}{\sqrt{r}}\)（不改变总面积）

我们将预设 n 个不同的 size 与 m 个不同的 ratio，并最终生成 \(nm\) 个锚框。

对于编号为 i 的锚框，我们根据以下条件选择使用的参数：

\[ \left\{ \begin{align*} size[i]\ and\ ratio[0],\ i \leq n \\ size[0]\ and\ ratio[i-n],\ i \gt n \end{align*} \right. \]

N+1 分类器

对于每一个锚框，SSD 使用 3x3 Conv 来实现 N+1 多分类器，同时采用 padding=1 使得输入输出大小一致。

输出的 \(channel = n\_anchor * (n\_class+ 1)\)，每一个通道对应了编号为 i 的锚框对物体归属于 n+1 类中每一个字累的置信度。

  • 假设输出为 \(Y\)，则第 \(n\) 个输入样本的第 \((i,j)\) 个像素的置信度为 \(Y[n,:,i,j]\)

  • 对于以像素 \((i,j)\) 为中心生成的第 \(a\) 个锚框：

    • channel a * (n_class + 1) = 是背景的置信度
    • channel a * (n_class + 1) + 1 + b = 包含第 b 种物体的置信度

预测真实边框

真实边框可以是 任意形状，我们可以用四元组描述 锚框 \(\rightarrow\) 真实边框 的变换过程 => 输出的总通道数为 n_anchor * 4。

同样通过一次 3x3 Conv + padding=1 的卷积层实现

  • 假设总输出为 \(Y\)，则样本 \(n\) 中像素 \((i,j)\) 对应的四元组为 \(Y[n,:,i,j]\)
  • 对于第 \(a\) 个锚框，其变换存储在通道 [a*4, a*4+3] 中

减半模块

使用 Conv Block 将输入的长宽减半，从而获取多尺度的预测。

该模块由 两个 Conv->BatchNorm->ReLU 构成，使用 3x3 卷积 + padding=1 做同 shape 变换，再通过 pool_size=2 的池化层实现折半。

Down Sample
def down_sample(n_filter):
    out = nn.HypbirdSequential()
    # 堆叠两个 Conv->BatchNorm->ReLU 模块
    for _ in range(2):
        out.add(
            nn.Conv2D(n_filter, 3, strides=1, padding=1),
            nn.BatchNorem(in_channel=n_filter),
            nn.Activation('relu')
        )
    # 使用 MaxPool 折半
    out.add(nn.MaxPool2D(2))

    return out

合并不同 Scale 的预测输出

SSD 会在不同的 Scale 同时进行预测，由于不同层中的长宽和锚框选择不同，各层输出的 shape 也各不相同。

此处以多分类器处理为例，边框预测同理：

因为第一个纬度（样本数）在各层保持不变，因此我们将 channel 移动到最后一个纬度，并进行 Flatten 操作。随后就可以在第二个纬度上直接 concat 各层输出：

def flatten_pred(pred):
    # 把 channel 挪到最后、flatten
    return pred.transpose(axes=(0,2,3,1)).flatten()

def concat_pred(preds):
    return nd.concat(*preds, dim=1) # 都是二维数组（且第一纬度一致），直接拼接

Non Maximum Suppression

由于我们对每个 pixel 都会生成数个锚框，最后的预测结果中会存在大量相似的表框。

思路：若两个锚框的 IoU 相近，我们只保留置信度较高的那一个。

误差计算

IoU 交集除并集

我们一般使用 Jaccard 距离判断两个集合的相似度：

\[ J(A,B) = \frac{|A \cup B|}{|A \cap B|} \]

对于两个区域，我们可以定义 Intersection over Union，IoU 越大表示两个区域越相似

\[ IoU = \frac{S_{相交区域}}{S_{相并区域}} \]
Loss Function

SSD 生成锚框中的 大多数 并不能框住感兴趣的物体 => 这些无效框与 任意有效框 的 IoU 均小于某个阈值。

对于这类锚框：

  • 我们不应在损失函数中考虑，因为它们没有对应真实边框

  • 由于这些框远多于有效框，我们可以保留其中最接近有效框的部分

    => 使用 mask 参数屏蔽不需要惩罚的样本

由于对边框的预测也是一个回归问题，我们也可以选择：

  • L2 损失函数 \(f(x) = x^2\)：对较大的误差惩罚很高
  • L1 损失函数 \(f(x) = |x|\)：缺点是在 x=0 处不唯一，可能影响收敛

  • 平滑L1 损失函数：

    \[ f(x) = \left\{\begin{align*} \frac{(\sigma x)^2}{2} &\ \ if\ x \lt \frac{1}{\sigma^2} \\ |x| - \frac{1}{2\sigma^2},&\ \ otherwise \end{align*} \right. \]

RNN 循环神经网络

RNN 的预测结果是一个 “序列”

  • CNN 属于 前馈神经网络，Sequential 的串联架构使得输出只能沿链前向传递。

    符合 马尔可夫链 的假设：当前状态仅依赖于上 一个 状态

  • 在 RNN 中，当前层的输出不仅直接传递给下一层；同时还会输出一个 隐藏状态，在当前层处理 下一个样本时 使用

    当前状态依赖于前面 所有状态，适用于 前后有依赖关系的样本（比如 NLP 问题）

    由于 Ht 依赖于 Xt & Ht-1，因此不能进行 并行计算

graph LR
    A(Input)-->B[[Layer]]-->C(Output)
    B--"隐藏状态"-->B
\[ \begin{align*} H_t &= \Phi(X_tW_{x\times h} + H_{t-1}W_{h\times h} + b_{1 \times h})\\ \hat{Y_t} &= softmax(H_tW_{h\times y} + b_{1 \times y}) \end{align*} \]

在语言模型中，特定序列 \((w_1,w_2,...,w_t)\) 出现的概率为：

\[ P(w_1,w_2,...,w_t) = P(w_1) \cdot P(w_2 | Ww1) \cdot P(w_3 | w_1w_2) ... P(w_t|w_1w_2...w_{t-1}) \]

定义模型

RNN 使用了 按元素操作的 双曲线正切函数作为激活函数，值域为 \([-1,1]\)：

\[ tanh(x) = \frac{1-e^{-2x}}{1+e^{-2x}} \]

梯度裁剪

在 RNN 的训练过程中，当每个时许训练样本的 seq_len 过大/过小 时，目标函数关于 t 时刻 隐藏层 的梯度就容易 衰减 Valishing / 爆炸 Explosion

当梯度过大时，我们可以尝试将其 投影 至一个较小的尺度上：假设所有梯度拼接得到的向量为 \(g\)，裁剪阈值为 \(\theta\)，有

\[ g = min(\frac{\theta}{\|g\|}, 1) \cdot g \Rightarrow \|g\| \lt \theta \]
def grad_clipping(params, theta):
    norm = nd.array([0.0])
    for p in params:
        norm += nd.sum(p.grad ** 2)
    norm = nd.sqrt(norm).asscalar()
    if norm > theta:
        for p in params;
            p.grad[:] *= theta / norm

LSTM 长短期记忆

成功解决 梯度衰减 问题

RNN 会将 有关&无关 信息全部记住

LSTM 通过 memory cell 进行选择性记忆：过滤噪声、留存重要信息，从而减轻记忆负担

前向传播过程

每个 memory cell 均有 三个 input：

  1. \(c^{i-1}\)：记忆前部序列中的重要信息（支持记忆较远处的信息）
  2. \(h^{i-1}\)：普通 RNN 也有，所有 前序信息之和，距离越远信息丢失越多
  3. \(x^i\)：当前时间点的 feature

单个 memory cell 的结构
memory cell 中三个 gate 的作用如下：

  • 遗忘门：选择性丢弃 \(h^{i-1}\) 和 \(x^i\) 中不重要的部分
  • 输入门：尝试向长期记忆 \(c^{i-1}\) 中塞入新的信息
  • 输出门：筛选 \(h^i\) 中实际需要输出的信息
